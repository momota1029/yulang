# instant（時刻値）仕様

決定日: 2026-07-02
著者: Claude (Fable 5) — ユーザ委任（「instant の仕様も欲しい。分かりやすいのでお任せします」）に基づく。
状態: **仕様（authoritative）**。意味論は stable contract、綴りは provisional spelling
（[2026-07-01-stable-standard-api.md](2026-07-01-stable-standard-api.md) の語彙に従う）。

目的は一つ: `file_meta.modified` のような **host 由来の時刻メタデータ**と、
その素朴な比較・差分に型を与えること。カレンダー・タイムゾーン・書式・
タイマーは全て非目標であり、この仕様の上に将来別レイヤとして載せる。

## 1. 中心決定

1. **instant はタイムライン上の一点である。** タイムゾーンも暦も持たない純データ。
   「いつ」であって「何時何分」ではない。表示や暦への変換は将来の time レイヤの仕事。
2. **表現は Unix epoch からの経過ナノ秒の int（公開）。**

   ```yu
   pub struct instant { epoch_nanos: int }     -- 綴り provisional
   pub struct duration { nanos: int }
   ```

   隠す理由がないので純データとして公開する（path・書記素と同じ「素朴な表現主義」）。
   int の範囲は epoch ±約 292 年であり、当面十分。範囲設計を変える場合は本仕様を改訂する。
3. **instant は host act 経由でしか生まれない。**

   ```yu
   pub host act clock:
       pub now: () -> instant                  -- sync tier
   ```

   純粋関数で「現在時刻」を作る道は存在しない。これは様式ではなく、
   record/replay（全ての非決定性が host act registry を漏斗として通る）と
   決定的並列性の前提を守る整合性条件である。リテラルと
   `instant { epoch_nanos: n }` 構築は再生・fixture のために許す
   （構築は決定的なので漏斗を破らない）。
4. **演算は最小限。**
   - `instant - instant = duration`
   - `instant + duration = instant`（`- duration` も同様）
   - `duration ± duration = duration`
   - instant / duration とも `Eq` / `Ord` を impl する。
5. **壁時計であり、単調性は保証しない。** host の時計が戻れば `now()` も戻る。
   経過時間の計測（単調時計）は**別の operation** として将来 `clock` act に足す
   （綴り未定。instant と混ぜない）。docs には「instant を性能測定に使わない」と書く。
6. **うるう秒は unspecified。** host の時計観をそのまま映す。仕様は関与しない。
7. **表示は二系統**（Show vs Debug の既存方針に従う）:
   - Show: RFC 3339 UTC（例: `2026-07-02T12:34:56.789Z`。秒未満は必要桁のみ）。
   - Debug: 構造どおり（`instant { epoch_nanos: ... }`）。
8. **`file_meta.modified: opt instant`。** missing / denied / host が時刻を
   持たない場合は `nil`。例外にはしない（meta は失敗しない、の一貫）。

## 2. 使用例（規範例）

```yu
my m = file::meta "notes.txt"
case m.modified:
    opt::just t ->
        if clock::now() - t > days 30:       -- days は duration の糖衣（将来）
            say "古いファイルだよ"
    opt::nil -> ()
```

`days` / `secs` などの duration 糖衣は便利 wrapper として後で足してよいが、
中心 API とは呼ばない。

## 3. 非目標（この仕様が縛らないと明言する場所）

- カレンダー（年月日への分解）、タイムゾーン、locale、書式付き parse / format。
  すべて instant の上に純 Yulang で載せる将来レイヤ。
- タイマー・sleep・デッドライン。suspend tier の operation が要るため、
  scheduler / structured concurrency の決定後に扱う。
- 単調時計（決定5 のとおり別 operation）。

## Rollback 条件

- instant にタイムゾーンや暦のフィールドを足したくなったら停止（それは別レイヤ）。
- `now` を registry 漏斗の外（純粋関数・暗黙変数）で提供したくなったら停止
  （record/replay と決定的並列性の前提が壊れる）。
- file 工事の都合でこの仕様を拡張したくなったら、拡張せず file 側を延期する。
