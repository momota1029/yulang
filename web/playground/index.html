チートシート

          

            Playground は std::prelude と std::undet::* を暗黙 import している。
          

        

        

          

            
              

                Group
                Form
                Notes
              

            
            
              

                値
                1, 2.0, true, "text"
                整数、浮動小数、真偽値、文字列。
              

              

                束縛
                my x = 1 + 2
                ローカルな名前を作る。
              

              

                公開束縛
                our f x = x + 1, pub answer = 42
                our と pub は型推論結果の Types 欄にも出る。説明用の小さな API を見せるときに便利。
              

              

                参照
                my $x = 10, $x, &x = 11
                $ は読み取り、& は書き込み先。普通の変数再代入ではなく、参照の読み書きを明示する。
              

              

                関数
                my id x = x, \x -> x + 1
                名前付き関数とラムダ。
              

              

                block
                ({ my x = 1; x + 1 })
                複数の文をまとめて最後の式を値にする。インデント block と brace group のどちらでも書ける。
              

              

                条件分岐
                if x == 1: "one" else: "other"
                条件で値を選ぶ。
              

              

                パターン
                case xs: [] -> 0; [x,..xs] -> x
                リスト、enum、record を分岐できる。
              

              

                リスト
                [1, 2, 3], [..xs, x]
                リスト literal と spread。
              

              

                リスト関数
                map(xs, f), filter(xs, pred), sort(xs)
                empty, singleton, cons, uncons, fold, rev, first, last もある。
              

              

                加算
                1 + 2, "a" + "b", [1] + [2]
                Add role。int、float、str、list で使える。
              

              

                比較
                x == y, x <= y
                Eq と Ord role。int と float が主な対象。
              

              

                長さ
                [1, 2, 3].len, "abc".len, len xs
                Len role。list と str に実装されている。method と関数呼び出しのどちらでも使える。
              

              

                範囲
                1..3, 1..<3, ..3, 1..
                range 値を作る。list/str index と組み合わせる。有限範囲は Fold でも使える。
              

              

                range fold
                (((0..<5).fold) 0) (\acc i -> acc + i)
                有限範囲を int の列としてたたむ。この例は 0 + 1 + 2 + 3 + 4。
              

              

                index
                xs[0], xs[1..3], s[0], s[1..3]
                Index role。xs.index 0 のような method 呼び出しもできる。
              

              

                opt
                nil, just x
                first, last, uncons, .once などが返す。constructor は prelude からそのまま使える。
              

              

                非決定
                each [1, 2, 3]
                playground では std::undet::* を暗黙 import している。
              

              

                guard
                guard (x == 3)
                std::undet::guard は条件が false のとき fail() する。each と組み合わせて候補を絞る。
              

              

                非決定 handler
                (each [1, 2] + each [4, 5]).list
                .list は全解、.logic は探索順、.once は最初の解。
              

              

                junction
                if all [1, 2, 3] < any [2, 3, 4]: 1 else: 0
                any と all で比較式を分岐させる。
              

              

                record
                { width: 1, height: 2 }
                pattern では { width = 1, ..rest } のように default/spread も使える。
              

              

                省略可能引数
                my area({width = 1, height = 2}) = width * height
                record pattern の default を使う。area { width: 3 } や area {} のように呼べる。
              

              

                struct with
                struct point { x: int } with: our p.double = p.x + p.x
                データ型と method をまとめて定義する。point { x: 3 }.double のように呼べる。
              

              

                enum / case
                enum color = red | blue, case c: color::red -> 1
                constructor pattern で分岐する。payload つき variant も扱える。
              

              

                role
                role Display 'a: our a.show: str
                impl Display int: our x.show = ... のように method を足す。
              

              

                effect
                act io: our read: () -> int
                catch io::read(): io::read(), k -> k 1 のように捕まえる。
              

              

                catch
                catch action: effect::op(x), k -> k x
                effect operation と継続 k を受け取って処理する。捕まえない effect は外側へ残る。
              

              

                do
                my x = f do
                Haskell の do 記法ではなく、同じ block の後続計算を差し込む marker。標準ライブラリの制御関数で使う。
              

              

                for control
                for x in xs: if x == 2: next else: ()
                for は内部で std::flow::loop::for_in を使う。last は break、next は continue、redo は同じ要素をやり直す。
              

              

                sub return
                sub: for x in 0..: if x == 5: return x else: ()
                sub: は prefix の return x を捕まえて値 x にする。無限範囲や fold の途中脱出を値として扱える。
              

            
          

        

      

      

        

          
Cheat Sheet

          

            The playground implicitly imports std::prelude and
            std::undet::*.
          

        

        

          

            
              

                Group
                Form
                Notes
              

            
            
              

                Values
                1, 2.0, true, "text"
                Integers, floats, booleans, and strings.
              

              

                Bindings
                my x = 1 + 2
                Create a local name.
              

              

                Public bindings
                our f x = x + 1, pub answer = 42
                our and pub also appear in the Types pane, which is useful for small examples.
              

              

                References
                my $x = 10, $x, &x = 11
                $ reads and & writes. Reference access is explicit instead of normal reassignment.
              

              

                Functions
                my id x = x, \x -> x + 1
                Named functions and lambdas.
              

              

                Blocks
                ({ my x = 1; x + 1 })
                Group statements and use the final expression as the value. Indented blocks and brace groups both work.
              

              

                Conditionals
                if x == 1: "one" else: "other"
                Choose a value by condition.
              

              

                Patterns
                case xs: [] -> 0; [x,..xs] -> x
                Branch on lists, enums, and records.
              

              

                Lists
                [1, 2, 3], [..xs, x]
                List literals and spread.
              

              

                List functions
                map(xs, f), filter(xs, pred), sort(xs)
                empty, singleton, cons, uncons, fold, rev, first, and last are also available.
              

              

                Addition
                1 + 2, "a" + "b", [1] + [2]
                The Add role works for ints, floats, strings, and lists.
              

              

                Comparison
                x == y, x <= y
                The Eq and Ord roles mainly target ints and floats.
              

              

                Length
                [1, 2, 3].len, "abc".len, len xs
                The Len role is implemented for lists and strings. Method and function-call forms both work.
              

              

                Ranges
                1..3, 1..<3, ..3, 1..
                Create range values. They combine with list/string indexing, and finite ranges can also use Fold.
              

              

                Range fold
                (((0..<5).fold) 0) (\acc i -> acc + i)
                Fold a finite range as an int sequence. This example computes 0 + 1 + 2 + 3 + 4.
              

              

                Index
                xs[0], xs[1..3], s[0], s[1..3]
                The Index role also supports method calls like xs.index 0.
              

              

                Opt
                nil, just x
                Returned by first, last, uncons, .once, and similar APIs.
              

              

                Nondeterminism
                each [1, 2, 3]
                The playground implicitly imports std::undet::*.
              

              

                Guard
                guard (x == 3)
                std::undet::guard calls fail() when the condition is false, so it can filter candidates from each.
              

              

                Nondet handlers
                (each [1, 2] + each [4, 5]).list
                .list collects all solutions, .logic keeps search order, and .once returns the first solution.
              

              

                Junctions
                if all [1, 2, 3] < any [2, 3, 4]: 1 else: 0
                any and all branch comparison expressions.
              

              

                Records
                { width: 1, height: 2 }
                Patterns can use defaults and spread, such as { width = 1, ..rest }.
              

              

                Optional args
                my area({width = 1, height = 2}) = width * height
                Use record-pattern defaults. Calls like area { width: 3 } and area {} work.
              

              

                struct with
                struct point { x: int } with: our p.double = p.x + p.x
                Define a data type and its methods together. Call it like point { x: 3 }.double.
              

              

                enum / case
                enum color = red | blue, case c: color::red -> 1
                Branch with constructor patterns. Variants with payloads are supported too.
              

              

                Roles
                role Display 'a: our a.show: str
                Add methods with implementations like impl Display int: our x.show = ....
              

              

                Effects
                act io: our read: () -> int
                Catch operations with handlers such as catch io::read(): io::read(), k -> k 1.
              

              

                catch
                catch action: effect::op(x), k -> k x
                Handle an effect operation and its continuation k. Unhandled effects remain outside.
              

              

                do
                my x = f do
                Not Haskell do-notation. It marks where the rest of the current block is passed into a control function.
              

              

                for control
                for x in xs: if x == 2: next else: ()
                for uses std::flow::loop::for_in. last breaks, next continues, and redo retries the same element.
              

              

                sub return
                sub: for x in 0..: if x == 5: return x else: ()
                sub: catches prefixed return x and turns early exit into value x.
              

            
          

        

      

      

        

          
言語ガイド

          

            Yulang は型推論、role、代数的 effect を中心にした実験的な言語です。
            この playground で試すときに特に重要な見方を短くまとめています。
          

        

        

          

            
読む順番

            

              まずは左の editor で小さな式を書き、Result と Types を見ます。
              our や pub の binding は Types に推論結果が表示されます。
            

            
our twice x = x + x
twice 21

          

          

            
型と role

            

              + や <= は固定の組み込み演算ではなく、prelude の role method として解決されます。
              多相な binding では Add<α> のような requirement が型に残ります。
            

            
our twice x = x + x
// twice: Add<α> => α -> α

          

          

            
制御は effect で見える

            

              sub:、return、for、last は、標準ライブラリの effect handler と専用の lowering がつながって動きます。
              Examples の Effects では、handler が特定の effect だけを消して外側の effect を残す型を見られます。
            

            
sub:
    for x in 0..:
        if x == 5: return x
        else: ()
    0

          

          

            
参照は明示的

            

              $x は read、&x = value は write です。
              通常の変数再代入ではなく、局所的な effect scope として実装されています。
            

            
{
    my $x = 10
    &x = $x + 1
    $x
}

          

          

            
非決定性

            

              each は候補を選び、.list や .once が計算を実行します。
              playground では std::undet::* が暗黙に使えます。
            

            
(each [1, 2, 3] + each [4, 5, 6]).list

          短い英語まとめ
          

            Yulang is an experimental expression-oriented language built around type inference,
            structural types, roles, and algebraic effects. Many control-flow features are
            expressed by the standard library, while syntax such as sub:,
            for, $x, and effectful if conditions connect
            user code to those library-defined handlers.
          

        

      

      

        

          
Language Guide

          

            Yulang is an experimental language centered on type inference, roles,
            and algebraic effects. This guide highlights the parts that matter most
            while trying it in the playground.
          

        

        

          

            
Reading order

            

              Start with a small expression in the editor, then inspect Result and Types.
              Bindings marked with our or pub show inferred types in Types.
            

            
our twice x = x + x
twice 21

          

          

            
Types and roles

            

              Operators like + and <= are resolved as prelude role methods,
              not as fixed built-ins. Polymorphic bindings can keep requirements such as
              Add<α> in their inferred types.
            

            
our twice x = x + x
// twice: Add<α> => α -> α

          

          

            
Control via effects

            

              sub:, return, for, and last are connected
              to standard-library effect handlers through dedicated lowering.
              The Effects example shows a handler removing one effect while leaving others.
            

            
sub:
    for x in 0..:
        if x == 5: return x
        else: ()
    0

          

          

            
Explicit references

            

              $x reads and &x = value writes.
              This is implemented as a local effect scope rather than normal variable reassignment.
            

            
{
    my $x = 10
    &x = $x + 1
    $x
}

          

          

            
Nondeterminism

            

              each chooses candidates, and .list or .once
              runs the computation. The playground implicitly imports std::undet::*.
            

            
(each [1, 2, 3] + each [4, 5, 6]).list
