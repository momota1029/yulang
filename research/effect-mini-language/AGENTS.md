# Effect Mini Language Agent Notes

この directory は、Yulang 本体の実装仕様ではなく、論文調のミニ言語を使った論考作業の場所である。

## 目的

- 効果・重み・filter・floor・stack 操作を、実装から切り離した小さい計算体系として整理する。
- Yulang 本体の現行仕様と、論考用 toy language の規則を混同しない。
- 実装へ急がず、定義、例、反例、補題、未解決点を分けて扱う。

## 文体

- 論考本文は、必要なら論文調の技術文体で書く。
- 会話用のくだけた口調を成果物へ混ぜない。
- 日本語と英語のどちらで書いてもよいが、一つの節では文体を揃える。

## 記法

- 記法は勝手に増やさない。
- 記法を変えるときは、既存の定義、例、式変形への影響を確認する。
- `filter`、`floor`、`pop`、`push`、stack weight、effect family の意味を曖昧にしない。
- Yulang の surface syntax と、論考用の abstract syntax を区別する。

## 作業の進め方

- まず小さい例を置き、そこから一般規則を抜き出す。
- 直感だけで等式を書かず、どの合成規則を使った変形かを明示する。
- 失敗例や違和感のある例は消さず、反例または open question として残す。
- 実装案を書く場合も、本文の主張と分けて「implementation note」として置く。

## Typesetting

- タイプセットには `latexmk-diff-head [拡張子抜きのファイル名]` を用いる。
- 例: `latexmk-diff-head draft`
- `latexmk` を直接呼ぶ前に、この directory の規約と差分確認の意図に反していないかを見る。

## 禁止事項

- Yulang 本体仕様として確定していない規則を、確定仕様のように書かない。
- 実装の都合だけでミニ言語の規則を曲げない。
- 反例を見つけたときに、用語だけを変えて問題を隠さない。
- 検索しやすさを壊す長すぎるファイル名や、意味の薄い `misc` / `notes` ファイルを増やさない。
