個人で設計・実装している実験的プログラミング言語 Yulang の型推論について、主型性の定式化と既存研究上の位置づけをご相談したく、ご連絡いたしました。先生が、高階関数を含む体系での主型推論を研究されていることから、本件についてご意見を伺いたいと考えました。

Yulang は代数的エフェクトとハンドラを備え、値型と effect row を subtyping 制約から同時に推論します。現在、次のような設計を実装し、簡略化した中核言語上で形式化を進めています。

1. 具体的な値型や effect family 集合を注釈しなくても、高階関数について値型と latent effect を一般化した公開型を推論する
2. handler combinator の返り値型と residual effect row を一般化する
3. annotation は具体的な値型を決めず、主に関数の形と effect の可視性・流出を局所的に制約する

内部では、subtyping 制約に方向付きの stack weight を付け、handler が effect row から除去できる部分を、その境界から可視な effect に限定しています。

現在特に判断に迷っているのは、次の二点です。

- 主型性を内部の weighted scheme に対して述べるべきか、公開型への射影後にも述べられるか
- この annotation 設計を、既存の qualified typing や row-polymorphic effect system の中でどう位置づけるべきか
