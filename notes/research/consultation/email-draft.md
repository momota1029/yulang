現在、個人で設計・実装している実験的プログラミング言語 Yulang において、algebraic effects / handlers に近い機構を含む型推論系を作っています。特に、既存研究との関係や研究対象としての見込みについて判断がつかず、ご相談できないかと思いご連絡いたしました。

Yulang では、値型と effect row を subtyping 制約から同時に推論します。現在の設計では、次のような性質を実装上で観察しています。

- 通常の式・高階関数を含め、値の型と effect は原則として推論される
- effect の導入や呼出し元への伝播には、effect annotation を要求しない
- handler によって effect を局所化・消去する境界に限り、対象となる effect を明示的な契約として与える
- この明示は単なる型推論補助ではなく、effect の消去を許可する局所的な契約として働く

内部では、subtyping 制約に方向付きの stack weight を付け、handler が effect row から除去できる部分を、その境界から可視な effect に限定しています。自分の理解では、「effect の導入・伝播は推論し、handler による消去だけを明示契約として check する」という非対称な型付け規律になっているのではないかと考えています。

ただし、algebraic effects、effect typing、handler typing、row polymorphism 等の既存研究との正確な差分を十分に判断できていません。もし可能でしたら、研究として見る価値がありそうか、また見るならどの観点から整理すべきかについて、短時間でもご助言いただけないでしょうか。
