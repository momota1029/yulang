## infer local recursive effect passthrough

- local recursive binding の scheme finalize で、再帰本体が環境側の effect residual を通すケースを追跡。
- `my loop b = if b: loop (judge (f ())) ()` のような局所再帰で、callback effect が `any` に潰れず、外側の `h` に `(() -> ['a] any) -> ['a] ()` として残ることを確認。
- `for_in` 系の注釈付き local loop は、注釈あり引数を passthrough 対象にしないようにして、std check のハングを避けた。
- bound machine に live variable adjacency を持たせ、scheme の root effect var から環境へ到達する代表を拾えるようにした。

## labeled sub lambda の local scheme

- `\sub 'inner x -> 'outer.return x` の local scheme が `any -> never` に落ち、
  外側ラベルの return effect が消える問題を追跡。
- local binding の RHS で出た subtype constraints を、scheme finalize 前に drain して、
  同じ block の後続文が読む local scheme に閉包済みの制約を反映するようにした。
- surface `sub` / labeled `sub` は、body effect から handler が扱う return effect を除き、
  残りを結果 effect へ通す制約を lowering 時点で明示するようにした。
- act operation の参照は signature から value へ subtype 制約を張る。ただし現在の bare
  `return` は既存の `current_sub_return_target` 経路で扱うため、この immediate 制約からは外す。
