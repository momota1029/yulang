## labeled sub lambda の local scheme

- `\sub 'inner x -> 'outer.return x` の local scheme が `any -> never` に落ち、
  外側ラベルの return effect が消える問題を追跡。
- local binding の RHS で出た subtype constraints を、scheme finalize 前に drain して、
  同じ block の後続文が読む local scheme に閉包済みの制約を反映するようにした。
- surface `sub` / labeled `sub` は、body effect から handler が扱う return effect を除き、
  残りを結果 effect へ通す制約を lowering 時点で明示するようにした。
- act operation の参照は signature から value へ subtype 制約を張る。ただし現在の bare
  `return` は既存の `current_sub_return_target` 経路で扱うため、この immediate 制約からは外す。

## specialize2 effect row payload merge

- playground std の list update で、同じ effect slot に `ref_update int` と
  `ref_update (list int)` が入ったとき、slot lower candidate 同士を同一 family という理由で
  invariant に payload 統合していたのが衝突原因だった。
- same-slot lower candidate の共存では effect row payload を即時 exact 比較せず、
  effect row merge / resolver 側で explicit union として残すようにした。
- resolver の explicit `Union` / `Intersection` は candidate join/meet で合流できない場合も、
  explicit 型として保持する。effect payload の `int | list int` を候補衝突として落とさないため。
- mono dump の nondet list/say テストは、method specialization 自体を見る assertion に弱めた。
  resolver の grouping 変更で括弧位置だけが変わるため、挙動の固定対象から外した。
