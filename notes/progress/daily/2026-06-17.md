## labeled sub lambda の local scheme

- `\sub 'inner x -> 'outer.return x` の local scheme が `any -> never` に落ち、
  外側ラベルの return effect が消える問題を追跡。
- local binding の RHS で出た subtype constraints を、scheme finalize 前に drain して、
  同じ block の後続文が読む local scheme に閉包済みの制約を反映するようにした。
- surface `sub` / labeled `sub` は、body effect から handler が扱う return effect を除き、
  残りを結果 effect へ通す制約を lowering 時点で明示するようにした。

  `return` は既存の `current_sub_return_target` 経路で扱うため、この immediate 制約からは外す。

## wildcard effect thunk / act method row boundary

- `[_]` の関数引数 effect 注釈は、通常の effect row 制約ではなく stack boundary を持つ fresh effect として lowering する。
  lambda lowering 側では関数の `arg_eff` に stack wrapper ではなく内側 effect var を出し、body/output 側で pop を持ち回る。
- specialize2 は def body が concrete な関数 arg effect signature を持つ lambda のときだけ、signature を先に与えて thunk 化する。
  OpenVar arg effect の lambda では signature seed を避け、未使用 effectful 引数が二重 force / make にならないようにした。
- `once` の合成 act method で `['a & [nondet; 'a]]` が `['a]` に落ちていた原因は、制約保存時に
  `[E; tail]` と `tail` を同じ tail という理由で subsume / prune していたこと。
  stack/floor の row-tail boundary を持つ source では row upper を保持し、row subtraction が明示的に reduced upper を保存した場合だけ stale row を prune するように分けた。

  `'a ['b & [std::control::nondet::nondet; 'b]] -> ['b] std::data::opt::opt 'a`

## parameter effect annotation の full effect / handler view 分離

- `std.control.nondet.nondet.#act-method:once` は shallow handler ではないため、
  `α [nondet; β] -> [β] opt α` として export されるべきだった。
- parameter の effectful annotation で、通常参照に使う full effect と、`catch x:` が使う
  stack-protected view を分けた。`x` を単に forward するだけなら full effect が結果に残り、
  handler と同じ `α & [handled; α] -> [α]` にはならない。
- caller-facing な `arg_eff` は concrete row では `full & row` とし、`catch` は full effect に
  handled row upper を張りつつ、stack view で subtraction hygiene を保つ。
- `[_]` の stack quantified handler annotation だけは、既存どおり lambda output の subtract
  id を持ち回る。具体 row annotation では output 全体に機械的な subtract を混ぜない。
- `StackWeight` は `floor[Empty]` 後の concrete stack を吸収せず、後続の `Empty` stack だけを
  吸収するようにした。既存の pop まで concrete stack を残す性質を保つため。
