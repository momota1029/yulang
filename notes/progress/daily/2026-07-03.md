## Structured concurrency decision before serve

- Added `notes/design/2026-07-03-structured-concurrency-decisions.md` as the
  implementation decision for serve first slice.
- The decision fixes branch ownership and cancellation before any server
  in-process driver work:
  - `accept` resumes create child branches owned by the serving handler extent.
  - cancellation is queued through the scheduler as `Cancel { branch_id,
    reason }`, not as host-to-runtime reentry.
  - the first implementation slice only drops suspended branches immediately;
    running-branch cooperative cancellation is a later slice.
  - branch drop means scope exit is not reached, so file managed lens updates
    rollback under the existing resource-lifetime contract.
- Updated `spec/2026-07-02-server-resource-api.md`,
  `spec/2026-07-02-io-resource-api.md`, `spec/README.md`, `docs/status.md`,
  `notes/todo/structured-concurrency.md`, and
  `notes/todo/contract-v1-file-resource-priority.md` to point at the new
  decision.

## Ambient failure typing blocked

- Tried the next obvious cleanup: make `file_buffer::ambient_get` return
  `result str io_err`, make `ambient_set` return `result unit io_err`, and have
  `file::text` unwrap those results.
- Rejected that change before commit. It made `std.io.file.text` project as:
  `path -> ref(file_buffer | io_err | '[io_err] <: file_buffer & io_err, str)`,
  which leaks an unsolved row constraint into the public signature. The mock
  unscoped fixture also failed with `conflicting type candidates` after changing
  the handler arms to resume with `result::ok`.
- Conclusion: native ambient read/write failure typing cannot be solved by
  locally wrapping `file_buffer` operations in `result`. It needs a real
  session/error boundary or a solver/lowering change that reuses the row
  variable cleanly before public projection. I restored the previous
  `str` / `unit` ambient operations and left the blocker open.
- Validation after revert:
  - `cargo fmt --all -- --check`
  - `cargo check -q -p evidence-vm -p yulang`
