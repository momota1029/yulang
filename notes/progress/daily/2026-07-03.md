## File buffer native ambient registration

- Added `std::io::file::file_buffer` to the runtime host manifest with
  `ambient_get` and `ambient_set` sync operations.
- Added a native handler-extent ledger for unscoped `file::text`. `ambient_get`
  reads once into the ledger, `ambient_set` updates the ledger, and successful
  run completion flushes dirty ambient buffers back to disk.
- Added `file_text_unscoped_native_handler_extent` to the `file-resource`
  contract. It proves the edit is visible through the unscoped `&` view, the
  backing file still has the old content during the handler extent, and the
  temp file is written at run completion.
