## Structured concurrency decision before serve

- Added `notes/design/2026-07-03-structured-concurrency-decisions.md` as the
  implementation decision for serve first slice.
- The decision fixes branch ownership and cancellation before any server
  in-process driver work:
  - `accept` resumes create child branches owned by the serving handler extent.
  - cancellation is queued through the scheduler as `Cancel { branch_id,
    reason }`, not as host-to-runtime reentry.
  - the first implementation slice only drops suspended branches immediately;
    running-branch cooperative cancellation is a later slice.
  - branch drop means scope exit is not reached, so file managed lens updates
    rollback under the existing resource-lifetime contract.
- Updated `spec/2026-07-02-server-resource-api.md`,
  `spec/2026-07-02-io-resource-api.md`, `spec/README.md`, `docs/status.md`,
  `notes/todo/structured-concurrency.md`, and
  `notes/todo/contract-v1-file-resource-priority.md` to point at the new
  decision.
