## Principal Substitution Candidates

ChatGPT へ渡すため、直近の実装結果とテスト結果をここにまとめる。

### Implemented

- `ApplyEvidence` に `substitution_candidates` を追加した。
- `complete_principal` が principal callee / arg / result slot から `PrincipalSubstitutionCandidate` を export するようにした。
- candidate は `var`、`relation` (`Lower` / `Upper` / `Exact`)、`ty`、`source_edge`、`path` を持つ。
- `ApplicationCallee` / `ApplicationArgument` の source edge と derived expected edge も candidate 入力にした。
- closed ではない structural candidate も evidence として残し、runtime solver 側だけが open candidate を無視する。
- `SubstitutionSpecialize` は spine 内の複数 `ApplyEvidence` から candidates を集め、次の場合だけ substitution にする。
  - `Exact` が 1 種類の closed type に一致する。
  - `Lower` と `Upper` の両方に同じ closed type がある。
  - 片側候補、衝突候補、open / `Any` / value `Never` は bind しない。
- core IR verbose 表示では、候補が `subst-candidates=[...]` として見える。

### Current Interpretation

- The candidate evidence plumbing works and reaches runtime specialization.
- It does not yet reduce `07_junction` pass count or specialization count.
- `fold_ints` is now split into two failure classes:
  - 4 apply sites still have no usable substitution/candidate evidence.
  - 4 apply sites have candidates, but only open principal-variable relations.
- The next design question is how to propagate open candidate relations across the apply spine, or how to connect accumulator/result slots so that `t1193` / `t1254` receive closed candidates.

## Reachability-Aware Substitution Skip Profile

ChatGPT の追加方針に従って、open-candidate graph solver へ進む前に
`SubstitutionSpecialize` の skip target が最終 `PruneUnreachable` 後も残るかを
profile に出すようにした。

### Implemented

- `SubstitutionSpecializeTargetSkips` に `survives_final_prune: Option<bool>` を追加した。
- monomorphize pipeline 完了後、最終 module に残った binding set で
  substitution-specialize の skip target を annotation するようにした。

  `survives_final_prune=yes/no/unknown` を表示するようにした。

### Interpretation

- `std::range::fold_ints` is pruned from `examples/07_junction.yu`.
- The open candidates for `t1193` / `t1254` are useful debug evidence, but they are
  not on a final surviving target in this benchmark.
- This supports parking the open-candidate graph solver for now.
- For `07_junction`, the remaining reachable substitution-specialize skip shown in
  the top targets is `std::list::&impl#187::fold__mono2`, and its reason is
  `skip-non-generic-callee`, not missing principal substitutions.

### Current Decision

- Do not strengthen the open candidate solver from
  `t <= t | u` / `t >= t & u` style relations yet.
- Treat `std::range::fold_ints` in `07_junction` as support noise unless a
  range-specific reachable example proves otherwise.
- If a range-specific example keeps `fold_ints` alive, prefer full-spine
  principal candidate generation from accumulator/result slots over open graph
  propagation.
- With runtime specialization now observable enough for this checkpoint, the next
  high-leverage area can be diagnostics unless a concrete surviving specialization
  target becomes noisy again.

## Actionable Substitution Skip Classification

Runtime specialization の checkpoint を閉じやすくするため、
`SubstitutionSpecialize` の skip target を actionable / benign に分けた。

### Implemented

- `SubstitutionSpecializeTargetSkips` に `actionable: bool` を追加した。
- `skip-non-generic-callee` / `skip-local-leaf-specialization` /
  `skip-recursive-leaf-specialization` は benign として扱う。
- それ以外の target skip は、principal evidence / handler evidence / adapter
  handling の改善余地があるため actionable として扱う。
- `--runtime-phase-timings` では skip target 行に `actionable=yes/no` を表示し、
  `skip_reachability surviving_actionable_targets=N surviving_benign_targets=M`
  も出す。
