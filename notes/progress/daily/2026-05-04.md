## Principal Substitution Candidates

ChatGPT へ渡すため、直近の実装結果とテスト結果をここにまとめる。

### Implemented

- `ApplyEvidence` に `substitution_candidates` を追加した。
- `complete_principal` が principal callee / arg / result slot から `PrincipalSubstitutionCandidate` を export するようにした。
- candidate は `var`、`relation` (`Lower` / `Upper` / `Exact`)、`ty`、`source_edge`、`path` を持つ。
- `ApplicationCallee` / `ApplicationArgument` の source edge と derived expected edge も candidate 入力にした。
- closed ではない structural candidate も evidence として残し、runtime solver 側だけが open candidate を無視する。
- `SubstitutionSpecialize` は spine 内の複数 `ApplyEvidence` から candidates を集め、次の場合だけ substitution にする。
  - `Exact` が 1 種類の closed type に一致する。
  - `Lower` と `Upper` の両方に同じ closed type がある。
  - 片側候補、衝突候補、open / `Any` / value `Never` は bind しない。
- core IR verbose 表示では、候補が `subst-candidates=[...]` として見える。

### Current Interpretation

- The candidate evidence plumbing works and reaches runtime specialization.
- It does not yet reduce `07_junction` pass count or specialization count.
- `fold_ints` is now split into two failure classes:
  - 4 apply sites still have no usable substitution/candidate evidence.
  - 4 apply sites have candidates, but only open principal-variable relations.
- The next design question is how to propagate open candidate relations across the apply spine, or how to connect accumulator/result slots so that `t1193` / `t1254` receive closed candidates.
