## Evidence VM rejected later grant execution shape profile

目的:

- `ProviderEnv` later-grant native close は復活させない。
- later-grant branch が fast path として遅くなった理由を、実行意味論を変えずに
  profile する。
- 見たいものは、later grant placement 探索コスト、仮に native close を有効にした
  場合の候補数、legacy / hypothetical native の continuation shape 差。

変更:

- `provider_env_later_grant_placement_profiled` を追加した。
- later-grant placement 探索の call / frame step / `Then` step / `ProviderEnv` step /
  `MarkerFrame` stop / found / not-found を記録するようにした。
- later-grant branch が現在の条件のまま native eligible になる件数を
  profile-only で記録するようにした。
- `close_marker_frame` した legacy continuation shape と、marker close を飛ばした
  hypothetical native continuation shape を記録するようにした。
- 実行意味論は変えていない。`ProviderEnvForeignLaterGrantCert` は入れていない。

読み:

- later grant の visible semantics は引き続き legacy と一致している。
- ただし placement 探索だけで nondet 10,794 frame steps、showcase 16,629 frame steps
  を踏んでいる。
- hypothetical native shape は marker frame を半分程度減らすが、`Then` と
  `ProviderEnv` の大きな形は残る。
- つまり、later grant branch が遅かった原因は close 条件の正しさではなく、
  placement 探索と continuation representation 側に寄っている可能性が高い。
- 次に触るなら later-grant native close の復活ではなく、ProviderEnv/Then
  continuation shape をどう軽くするかを見る。
