## `std::text::parse.word` の char specialization

- `word = label(some(satisfy(is_word)), "word")` は、主型としては
  `satisfy(char -> bool)` の反変引数から parse item が `char` の subtype として残る。
  `ParseError.unexpected` も associated `item` / `pos` に実 item / pos の supertype を要求するため、
  `item = char|'d` や `pos = 'b|'e` の残差は principal な情報として説明できる。
- 推論 core でこれを `char` に同一化すると、関数引数の反変性を弱めて principal 性を落とすので避けた。
- `word` 自体は標準の char parser として出したい値なので、alias ではなく
  `pub word(): [parse char 'err 'pos 'snap] list char = ...()` にした。
  direct alias annotation (`pub word: () -> ... = ...`) は temp 実験で `check-poly-std` が 35 秒 timeout したため採らない。
- surface は
  `() -> [std::text::parse::parse(char, 'a, 'b&'e, 'c)] list char where 'a: ParseError(item = char|'d, pos = 'b|'e)`
  まで戻った。associated 側の supertype 残差は仕様寄りで残す。

## string interpolation formatting protocol

- `spec/2026-06-06-syntax-design.md` に Rust 風の `%format{expr}` mini grammar を追記した。
  - kind は空/`?`/`x`/`X` を `Display` / `Debug` / `LowerHex` / `UpperHex` へ対応させる。
  - align/fill/sign/alternate/zero_pad/width/precision は `format_spec` に構造化して std protocol へ渡す。
- lowering 側で `StringInterpFormatText` を `FormatSpec` へ parse するようにした。
  - layout 指定なしの `%?{x}` / `%x{x}` / `%X{x}` は従来の `%{x}` と同じく direct selection
    (`.debug` / `.lower_hex` / `.upper_hex`) へ下げる。
  - layout 指定ありの `%04{x}` / `%#08x{x}` は
    `std::core::fmt::format_* (format_spec {...}) x` へ下げる。
  - invalid format text は引き続き `UnsupportedSyntax(StringInterpFormatText)` とする。
- std 側へ `format_kind` / `format_align` / `format_sign` / `format_spec` と
  `format_display` / `format_debug` / `format_lower_hex` / `format_upper_hex` を追加した。
  - `format_with_padding` の zero padding 分岐は、dump 上の式形が意図通り読めるよう途中値に分けた。
  - `+` sign は format 済み body が `-` で始まらない場合に sign prefix として反映する。
  - `prefix` は operator declaration keyword と衝突しやすいため、std 実装の内部名には使わず `head` にした。
- `dump-poly-std-in examples/showcase.yu std::core::fmt` で protocol 関数の surface 型を確認した。

## 計算を伴う変数取得の値制限実装

- `BindingFetch::{FetchValue, FetchComputation}` を追加した。
  - top-level binding は `finish_binding` 時に RHS の `Evaluation` から fetch 性質を記録する。
