## declared constructor の tuple payload 呼び出し

- `range::within (start, end)` のような spaced tuple group call が、expr 側では 1 個の
  tuple 引数として lowering され、declared constructor scheme 側の
  `bound -> bound -> range` と噛み合わず、wrapper の型に
  `range | (bound -> range)` が残っていた。
- pattern 側には既に arity に合わせて single paren group を展開する処理があったため、
  expr 側にも declared constructor に限定して同じ展開を追加した。
  通常の関数適用や非 constructor の tuple 引数には触れない。
- `std::data::range.{range,inclusive,exclusive,...}` は
  `int -> int -> range` / `int -> range` / `() -> range` まで戻った。

## 型注釈なし role default method と input variance

- `std::core::fmt::Display` / `Debug` の default method (`say`, `note`, `dd`) は
  method signature を持たないため、これまで role input variance 収集で role 全体を
  invariant に倒していた。
- その結果、`show_list_items(xs: list 'a) where 'a: Display` のような再帰関数で
  role predicate の lower / upper が別変数に分かれた時、
  `list('a & 'b) -> str where ('a | 'b): Display` が surface に出ていた。
- 型注釈なし default method は declared requirement ではないため、
  宣言 signature 由来の role kind variance を決める材料から外した。
  明示 signature を持つ method だけで variance を導出し、signature が一つもない role は
  これまで通り `Unused` から invariant 扱いへ落ちる。
- `std::core::fmt.show_list_items` / `debug_list_items` は
  `list 'a -> str where 'a: Display|Debug` まで戻った。

## same-path effect row item の compact merge

- `std::text::parse.choice` / `optional` / `label` の surface 型に
  `[parse ...] & [parse ...]` 相当の duplicate effect row が漏れていた。
  `parse` effect family と row tail 側の同一 path family が別々に finalize されていたことが原因。
- positive effect row の finalize で、`CompactType.cons`、`CompactRow.items`、
  row tail に出る rowish effect family を一つの `CompactRowItemMap` に集めてから
  `Pos::Row` へ戻すようにした。
  path が同じ row item は `merge_row_items(true, ...)` と同じ規則で payload を merge する。
- row item と constructor 側に同じ effect family path が共存する場合、payload 間の
  merge constraint も記録するようにした。
  併せて merge constraint key の interval shape に exact center var を入れ、
  複数 payload slot が同じ `Interval` key に潰れないようにした。
- `std::text::parse.choice` / `optional` / `label` は
  `[std::text::parse::parse 'b 'c 'd 'e]` の単一 row 表示まで戻った。
  `word` には payload/list 側の残差がまだ残るが、duplicate row 問題とは別筋。

## `std::text::parse.word` の char specialization

- `word = label(some(satisfy(is_word)), "word")` は、主型としては
  `satisfy(char -> bool)` の反変引数から parse item が `char` の subtype として残る。
  `ParseError.unexpected` も associated `item` / `pos` に実 item / pos の supertype を要求するため、
  `item = char|'d` や `pos = 'b|'e` の残差は principal な情報として説明できる。
- 推論 core でこれを `char` に同一化すると、関数引数の反変性を弱めて principal 性を落とすので避けた。
- `word` 自体は標準の char parser として出したい値なので、alias ではなく
  `pub word(): [parse char 'err 'pos 'snap] list char = ...()` にした。
  direct alias annotation (`pub word: () -> ... = ...`) は temp 実験で `check-poly-std` が 35 秒 timeout したため採らない。
- surface は
  `() -> [std::text::parse::parse(char, 'a, 'b&'e, 'c)] list char where 'a: ParseError(item = char|'d, pos = 'b|'e)`
  まで戻った。associated 側の supertype 残差は仕様寄りで残す。
