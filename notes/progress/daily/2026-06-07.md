## Monomorphize / runtime-lower debug

- `vm_host_open_in_uses_drop_flush` の `ref_update::update expected char got str` は、
  runtime-lower の expected evidence hint が `char|str` の value choice を片側へ
  投影していたことが直接原因だった。
  - expected arg / expected callee hint では、value choice leaf をそのまま runtime
    evidence に採用しない。
  - upper 側が具体的なら upper を使い、両側とも choice / imprecise なら `Unknown`
    へ落として構造だけ残す。
  - effect operation signature の payload var は、expected 側が value choice のとき
    署名側の var を保持する。
- `Fold::each` の graph は、role input を完全 closed とみなすまで待っていた。
  `ref_lines<t>` のように runtime head が見えている input は role selection /
  associated projection に使えるので、`role_input_ready` で待ち判定を緩めた。
- `std::undet::undet::each` の `Any -> never` surface は、monomorphic contextual
  apply が open named 引数を拒否していたため、`ref_lines<t>` から mono alias を
  作れていなかった。裸 var / Any / Unknown は拒否したまま、head のある open 型は
  contextual refinement に使う。

## Erased infer output first cut

- `yulang-erased-ir` crate を追加した。
  - `ErasedExpr` は node ごとの型・効果、apply evidence、annotation evidence、`Coerce` node を持たない。
  - binding / root expression は `{ typ, eff, ir }` の `InferredExpr` として持つ。
  - `RefExportTable.direct` / `resolved_typeclass` の所有者は erased IR 側に置いた。
- `yulang_infer::export_erased_program` を追加した。
  - legacy `TypedExpr` から `typed_ir::Expr` を経由せずに erased IR を作る bridge。
  - global `Var(DefId)` は synthetic `RefId` を発行し、`RefExportTable.direct` に戻す。
  - 既存 `ExprKind::Ref(ref_id)` は元の `RefId` を維持し、解決済みなら direct table に入れる。
  - `ExprKind::Coerce` は erased export で中身だけを出す。
- regression:
  - global def use が erased root 内で `RefId` になり、direct table に対応すること。
  - legacy typed export が coerce evidence を持つ canary でも、erased IR には `Coerce` が出ないこと。

確認:

- `cargo test -q -p yulang-erased-ir -p yulang-infer erased_ -- --nocapture`
- `cargo fmt --check`
- `cargo check -q -p yulang-erased-ir -p yulang-infer`

残り:

- first cut はまだ source lowering 本体の carrier を直接 `LoweredComputation` へ置き換えていない。
- destructuring / constructor payload 由来の binder は、まだ legacy `TypedPat` を経由している。
- `TypeClassObligation` は型だけ先に置いた。実際の obligation export は次の段階で扱う。

## Erased resolved role method refs

- infer 側に `ResolvedRoleMethodSelection` を追加し、role method selection の解決結果を
  `TypeVar -> { role, member, impl_member }` として保持するようにした。
  - `DeferredRoleMethodCall` 経由の解決と、通常の `.method` selection から role method へ
    解ける経路の両方で記録する。
  - `resolved_selections` の `DefId` だけでは role member と impl member の関係が失われるため、
    erased export 用の構造化 metadata として別 table にした。
- copied principal body の `TypeVar` 置換でも、resolved role method selection を
  ref projection と同じ責務で引き継ぐようにした。
- erased export は resolved role method call を `RefExportTable.resolved_typeclass` の
  synthetic ref に lower する。
  - `RefExportTable.direct` へ impl member を折り込まない。
  - export 側で `Index` / `index` などの文字列を推測しない。
- regression:
  - `impl Index int bool` の `1.index true` が erased IR 内の ref と
    `resolved_typeclass` entry を持つこと。
  - role member と impl member の identity が区別されること。

## Erased direct bind patterns

- direct binding header の pattern を `PatKind::UnresolvedName` ではなく
  `PatKind::As(Wild, def)` として lower するようにした。
  - preregister 済みの `DefId` が取れない場合だけ `UnresolvedName` に戻す。
  - `my x = ...` / `my f x = ...` の binder identity を lower 済み tree に残す。
- erased export は `As(Wild, def)` と、解決できる `UnresolvedName` を
  `erased::Pattern::Bind(def)` に落とす。
- regression:
  - block-local `my x = 1` が erased IR の let pattern で `Bind(def)` になること。
