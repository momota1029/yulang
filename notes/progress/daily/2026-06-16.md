## specialize2 thunk computation shape

- 推論で使う関数型 `['a] 'b` と、runtime/代入で使う `thunk['a, 'b]` を分ける方向で
  `specialize2` の emit 境界を見直した。
- `emit_expr` の内部返り値を `Expr` だけでなく `ComputationShape { effect, value }` 付きにした。
  - `[] thunk['a, 'b]` を force したら、以後は `['a] 'b` として持ち回る。
  - 後続の make は古い `actual` ではなく、この carried computation shape を `source` に使う。
  - これで `force` 後に stale `actual` で再度 `make` する二重 force/make を避ける。
- `catch` body も構文別の force 判定をやめ、現在の emitted shape が pure thunk value のときだけ force する。
- raw expression の値型を構文ごとに分けた。
  - constructor spine は raw mono では constructor value / partial constructor value を返すので、
    型上 effectful function に見えても thunk として force しない。
  - App / Select / Catch / Block などは「raw が返す値」と「計算としての effect/value」を分けて扱う。
- 境界注釈として mono へ出す `ForceThunk` / `MakeThunk` の source/target は runtime surface に閉じる。
  推論用の `arg_effect` / `ret_effect` を mono VM type に漏らさないため。

## for_in residual investigation

- `for_in` の `['e]` 消失は、単純な annotation lowering では再現しなかった。
  - `sub(x: [handled; 'e] _): ['e] ()` 経由では callback residual が残る。
  - `act loop` 内の `pub h(xs, f: _ -> [loop; 'e] _): ['e] _ = f xs` でも tail は再利用される。
  - `Fold.fold` role method selection 経由でも、明示 `sub` helper があれば residual は残る。
- したがって、残差消失は role selection 単体ではなく、`for_in` の
  `catch f x` → `redo.judge` → `next.sub` → `last.sub` のような handler chain で、
  scrutinee residual と handler body の current effect が同じ effect 変数へ合流するところに寄っている。
- 試したが戻した案:
  - `catch` の結果を `current handler effect + carried residual` に分け、
    `make_app` が effectful 引数を force するとき current だけを `arg_eff` に渡す案。
  - さらに `Block` / `if` へ split view を伝播する案。
  - どちらも full std / playground 回帰で timeout したため、今の制約 machine にそのまま載せるには重すぎる。
- 次にやるなら、`Computation.effect_view` へ局所的に情報を増やすより、
  ユーザー案どおり「推論用の effectful 型 `['a] 'b`」と
  「代入/force 用の `thunk['a, 'b]` 相当の型」を分け、force 時に current effect を小さい構造で引き回す。
  catch / handler chain 全体に split 制約をばらまかないこと。
- 確認:
  - `cargo check -q -p infer`
  - `cargo test -q -p infer lowering::tests::subtractable_helper_preserves_callback_residual_tail -- --exact --nocapture`
  - `cargo test -q -p infer lowering::tests::act_body_result_effect_annotation_reuses_callback_tail -- --exact --nocapture`
  - `cargo test -q -p infer lowering::tests::role_method_selection_preserves_callback_residual_tail -- --exact --nocapture`
