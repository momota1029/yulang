## specialize2 thunk computation shape

- 推論で使う関数型 `['a] 'b` と、runtime/代入で使う `thunk['a, 'b]` を分ける方向で
  `specialize2` の emit 境界を見直した。
- `emit_expr` の内部返り値を `Expr` だけでなく `ComputationShape { effect, value }` 付きにした。
  - `[] thunk['a, 'b]` を force したら、以後は `['a] 'b` として持ち回る。
  - 後続の make は古い `actual` ではなく、この carried computation shape を `source` に使う。
  - これで `force` 後に stale `actual` で再度 `make` する二重 force/make を避ける。
- `catch` body も構文別の force 判定をやめ、現在の emitted shape が pure thunk value のときだけ force する。
- raw expression の値型を構文ごとに分けた。
  - constructor spine は raw mono では constructor value / partial constructor value を返すので、
    型上 effectful function に見えても thunk として force しない。
  - App / Select / Catch / Block などは「raw が返す値」と「計算としての effect/value」を分けて扱う。
- 境界注釈として mono へ出す `ForceThunk` / `MakeThunk` の source/target は runtime surface に閉じる。
  推論用の `arg_effect` / `ret_effect` を mono VM type に漏らさないため。
