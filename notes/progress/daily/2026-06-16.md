## specialize2 thunk computation shape

- 推論で使う関数型 `['a] 'b` と、runtime/代入で使う `thunk['a, 'b]` を分ける方向で
  `specialize2` の emit 境界を見直した。
- `emit_expr` の内部返り値を `Expr` だけでなく `ComputationShape { effect, value }` 付きにした。
  - `[] thunk['a, 'b]` を force したら、以後は `['a] 'b` として持ち回る。
  - 後続の make は古い `actual` ではなく、この carried computation shape を `source` に使う。
  - これで `force` 後に stale `actual` で再度 `make` する二重 force/make を避ける。
- `catch` body も構文別の force 判定をやめ、現在の emitted shape が pure thunk value のときだけ force する。
- raw expression の値型を構文ごとに分けた。
  - constructor spine は raw mono では constructor value / partial constructor value を返すので、
    型上 effectful function に見えても thunk として force しない。
  - App / Select / Catch / Block などは「raw が返す値」と「計算としての effect/value」を分けて扱う。
- 境界注釈として mono へ出す `ForceThunk` / `MakeThunk` の source/target は runtime surface に閉じる。
  推論用の `arg_effect` / `ret_effect` を mono VM type に漏らさないため。

## for_in residual investigation

- `for_in` の `['e]` 消失は、単純な annotation lowering では再現しなかった。
  - `sub(x: [handled; 'e] _): ['e] ()` 経由では callback residual が残る。
  - `act loop` 内の `pub h(xs, f: _ -> [loop; 'e] _): ['e] _ = f xs` でも tail は再利用される。
  - `Fold.fold` role method selection 経由でも、明示 `sub` helper があれば residual は残る。
- したがって、残差消失は role selection 単体ではなく、`for_in` の
  `catch f x` → `redo.judge` → `next.sub` → `last.sub` のような handler chain で、
  scrutinee residual と handler body の current effect が同じ effect 変数へ合流するところに寄っている。
- 試したが戻した案:
  - `catch` の結果を `current handler effect + carried residual` に分け、
    `make_app` が effectful 引数を force するとき current だけを `arg_eff` に渡す案。
  - さらに `Block` / `if` へ split view を伝播する案。
  - どちらも full std / playground 回帰で timeout したため、今の制約 machine にそのまま載せるには重すぎる。
- 次にやるなら、`Computation.effect_view` へ局所的に情報を増やすより、
  ユーザー案どおり「推論用の effectful 型 `['a] 'b`」と
  「代入/force 用の `thunk['a, 'b]` 相当の型」を分け、force 時に current effect を小さい構造で引き回す。
  catch / handler chain 全体に split 制約をばらまかないこと。
- 確認:
  - `cargo check -q -p infer`
  - `cargo test -q -p infer lowering::tests::subtractable_helper_preserves_callback_residual_tail -- --exact --nocapture`
  - `cargo test -q -p infer lowering::tests::act_body_result_effect_annotation_reuses_callback_tail -- --exact --nocapture`

## playground list_update specialization fix

- `list_update` の残りは、`var_ref.update_effect` の `set:ref_update::update:get()` で、
  effect operation payload が値ではなく thunk として渡されることが原因だった。
  runtime では `&xs::set` の payload が nested `ref_update` thunk のまま逃げ、
  `resolve_ref_set_value` が list ではなく `6` を代入していた。
- effect operation application では、operation signature の `arg_effect` を payload thunk 判定に使わず、
  payload expression は `arg` の値 consumer で評価するようにした。
  payload 自体を thunk にしたい場合は、代入用 runtime type として `arg` が `thunk[...]` になるべきで、
  inference 側の open `arg_effect` をそのまま request payload 型へ持ち込まない。
- emit 側では、effect operation app の raw value type を runtime thunk value として扱うようにした。
  consumer が plain value なら force され、consumer が thunk なら make されるので、
  `[] thunk['a, 'b]` を force した後の current computation shape をそのまま持ち回れる。
- weighted pure upper は、weighted view の pure を slot の unweighted pure upper へ昇格しないようにした。
  stack weight で見た pure と元 slot 全体の pure は同じ意味ではないため。

## direct cast boundary calls

- `1 + 1.2` が `conflicting type candidates for slot 0: int vs float` になっていた件は、
  join 候補として direct cast を見る経路と、mono emit で boundary を `Coerce` にする経路が分かれていた。
- `specialize2` の boundary emit は、nominal `Con` 同士で direct cast rule がある場合、
  `Coerce` ではなく該当 `cast` def の mono instance を `Apply` する。
  `Coerce` は thunk make/force など runtime representation boundary の fallback として残す。
- `int -> float` 変換は `builtin_op::int_to_float` primitive として追加し、
  full std / compact playground std の `cast(x: int): float` の body がそれを呼ぶ。
  cast 自体は通常の `cast` 定義として解決・specialize される。
- compact playground std は `std/num.yu` を full std から include し、
  `std/num/frac.yu` も埋め込むようにした。
  これで `int -> frac -> float` と direct `int -> float` の両方を playground 側でも使える。

## raw computation cast boundary fix

- `point { x: 3, y: 4 }.norm2 + 1.12` は、method select の raw value `int` を
  computation shape の value `float` へ lift するところで旧 free `coerce_emitted_value` を通っていた。
  root expression boundary は cast def call へ直っていたが、raw computation lift だけが
  `Specializer2` の cast rule lookup を読めない経路に残っていた。
- `lift_raw_expr_to_computation` を `Specializer2` method へ移し、direct cast rule がある value boundary は
  `cast` def instance call として emit する同じ経路へ統一した。
- `run-control-std` は source key だけで control artifact cache を読むため、修正前に生成した
  `Coerce int => float` 入り artifact が同じ source set で残っていた。
  control IR の意味が変わったので `CONTROL_CACHE_FORMAT` を上げ、古い artifact を読まないようにした。
- playground regression の struct method case は、method 結果を float add に渡す形に更新した。
