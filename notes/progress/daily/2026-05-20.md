## Runtime control VM spike

- Added an experimental arena-backed control VM under `crates/yulang-runtime/src/vm/control.rs`.
  It still starts from lowered runtime IR and VM erasure, but compiles expressions to
  `ExprId` nodes before execution. Closures, thunks, resume continuations, and
  continuation frames carry `ExprId` plus an environment instead of cloning full
  runtime `Expr` trees into every closure/thunk/frame.
- Added hidden CLI entry `yulang debug control-vm [--print-roots] <path>` for
  source-to-runtime-to-control-VM experiments. This is intentionally not exposed as
  a user-facing run mode yet.
- The control VM preserves the real `std::undet` path: `each` still goes through
  `Fold`, `sub::sub` / `sub::return`, `branch` / `reject`, `BindHere`, `Thunk`,
  handler dispatch, and `Resume`. This replaces the earlier invalid hand-written
  loop model and measures the actual lowered runtime shape.
- Correctness checks:
  `use std::undet::*; (branch()).list` returns `[true, false]`;
  `use std::undet::*; (each [1, 2, 3] + each [10, 20]).list` returns
  `[11, 21, 12, 22, 13, 23]`;
  `use std::undet::*; (each 1..20 + each 1..20).list` returns the expected
  400-item list.
- Release timing on `/tmp/yulang_20x20_list.yu`
  (`use std::undet::*; (each 1..20 + each 1..20).list`, no root printing):
  existing VM `vm_eval` measured `387.499ms`; control VM `vm_eval` measured
  `41.409ms`. The control profile was `eval_expr_calls=41063`,
  `max_eval_depth=171`, `continuation_steps=12683`,
  `max_continuation_frames=73`.
- Verification:
  `cargo check -q -p yulang`, `cargo test -q -p yulang-runtime` passed before
  adding the focused regression, and
  `cargo test -q -p yulang-runtime control_vm_runs_std_undet_range_each_nested_sum_list`
  passed after adding the control VM regression.

## Control VM compact expression shape

- Removed the full runtime `Type` from every serialized `ControlExpr`. The
  interpreter only needed a small execution shape at runtime:
  whether a lambda parameter forces a thunk argument, whether a lambda/handler
  result should be wrapped as a thunk, and whether an apply should delay the
  argument. Those booleans are now compiled into `ControlExprKind` / closure /
  continuation frames. Full types remain only where execution needs the actual
  type payload, such as `AddId.allowed`, `Coerce.to`, and pattern metadata.
- Bumped the control VM artifact version to `1`, because this changes the
  serialized module shape.
- `/tmp/yulang_20x20_list_light_v1.ycvm` for
  `use std::undet::*; (each 1..20 + each 1..20).list` is 3.0K, down from the
  previous 14K binary artifact and the original 78K JSON artifact. A timed load
  run measured `control_vm_load=308.286us` and `vm_eval=30.623ms`.
- Hyperfine, release build:
  `target/release/yulang debug control-vm-load /tmp/yulang_20x20_list_light_v1.ycvm >/dev/null`
  measured `33.4 ms ± 0.6 ms`; the generated effects executable
  `/tmp/yulang_20x20_list_effects >/dev/null` measured `380.5 ms ± 15.7 ms`.
  The compact binary artifact load+eval path was `11.40 ± 0.52` times faster
  than the current generated native effects executable for this benchmark.

## Source compiled std artifact cache

- Added a CLI-side compiled-unit artifact cache for std source units. The source
  collection path now keeps the `SourceSet`, tries to read matching cached std
  `CompiledUnitArtifact`s, and lowers the entry source against imported std
  syntax / namespace / typed / runtime surfaces on cache hit.
- On cache miss, the CLI follows the existing full lowering path, finalizes the
  compact inference results before artifact construction, and writes std unit
  artifacts under the configured Yulang cache directory.
- Cached std runtime surfaces now carry primitive runtime dependencies whose
  paths are not normally owned by a source unit, such as `std::int::add`. The
  extra paths are deduplicated by emitted path, not by `DefId`, so canonical
  primitive paths survive even when the same definition is also visible through
  a unit-local alias. Unit-owned runtime paths such as `std::list::empty` remain
  in their owning unit and do not conflict during bundle merge.
- Release timing on `/tmp/yulang_20x20_list.yu`
  (`(each 1..20 + each 1..20).list`, control VM artifact emit):
  clean cache miss measured `lower=523.273ms`, with `parse=25.295ms`,
  `infer_lower=471.947ms`, and `std lower_roots=467.173ms`.
  cache hit measured `lower=26.094ms`, with `parse=0ns`, `infer_lower=0ns`,
  and `std_cache: hits=1`.
- The hit artifact loaded correctly with `control-vm-load --print-roots`.
  Remaining compile-time cost on the hit path is mostly collection and runtime
  dependency merging/lowering: the measured hit had `runtime_lower=125.796ms`
  and `monomorphize=87.254ms`, so dependency pruning is the next compile-time
  target if needed.
- Verification:
  `cargo check -q -p yulang`,
  `cargo test -q -p yulang-infer compiled_std_runtime_bundle_carries_non_unit_runtime_dependencies`,
  and
  `cargo test -q -p yulang-runtime control_vm_artifact_roundtrips_std_undet_range_each_nested_sum_list`
  passed.
