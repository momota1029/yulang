# 2026-05-18 Native Release Prep

## Completed

- Fixed the remaining recursive handler family that blocked a native release
  candidate:
  - recursive tuple-returning handler no longer leaks thunk/heap handles as
    pointer-looking integers;
  - recursive handler block-tail loop control no longer collapses to the local
    `5` value before the value arm / tuple construction finishes;
  - unit and bool root display keep user-facing value shape on covered native
    paths.
- Moved the fixed native handler notes into `notes/bugs/solved/` and updated
  the bug indexes.
- Refactored the large native CPS Cranelift runtime helper block into
  `crates/yulang-native/src/cps_repr_cranelift/runtime_i64.rs`, leaving the
  backend entry file easier to navigate.
- Split effectful direct-call reification into
  `crates/yulang-native/src/cps_effectful_calls.rs` and run it before capture
  inference, so recursive handler calls get an explicit resume continuation.
- Updated `README.md`, `docs/native-backend.md`, `docs/status.md`,
  `notes/design/native-remaining-failure-matrix.md`, and `tasks/current.md`
  for release prep.
- Added `docs/native-experimental-release.md` as the release-gate handoff note
  for the opt-in native subset.
- Fixed the N11/N13 release-gate regressions:
  - effectful direct-call reification now leaves direct calls alone when the
    result is immediately forced by the following statement or terminator;
  - ScopeReturn routing uses the handler's captured
    `return_frame_threshold` rather than re-deriving the boundary from a
    prompt_exit marker after frame slicing.

## Native GC Runtime Spike

- Added isolated `crates/yulang-native/src/gc_runtime.rs` as the first
  post-prototype runtime layout boundary.
- Covered `YValue` tagging, native object headers, traceable payloads,
  allocation helpers for all first-milestone object kinds, explicit root stack
  tracing, root frames and temporary-root helper scopes for helper
  temporaries, `YHeap` allocator boundary, allocation stress tracing, and i63
  overflow promotion to heap `BigInt`.
- Added allocated-heap stats and live root-trace reports to keep the spike heap
  and future MMTk-backed heap comparable without relying on debug printing.
- Extended the integer helper smoke surface from add-only overflow promotion to
  add/sub/mul and comparison across `i63` and current decimal `BigInt` spike
  values.
- Added monomorphic native layout descriptors, typed `NativeHeapBlock` storage,
  layout footprint/field offsets, and trace bitmaps. This records the intended
  post-monomorphization shape where `i64`/`u64` captures and tuple fields remain
  unboxed, only `YValue` fields are traced, and stack-frame layouts are not
  allocated as heap objects.
- Added a native layout registry and shared layout handles so repeated
  monomorphic heap blocks reuse one layout contract instead of carrying a full
  layout copy per object.
- Moved `NativeHeapBlock` field storage from an enum array to a raw payload
  buffer written and read through layout field offsets. `NativeFieldValue` now
  acts as construction/debug input rather than the stored representation.
- Added native symbols for static path and atom identity: `YSymbolTable`,
  compact `YSymbolId`, stable fingerprints, symbolic variants, and a `Symbol`
  native field lane for path/tag payloads that should not carry full path
  objects at runtime. Ruby-style atom strings are supported separately from path
  symbols.
- Added `YPerfectSymbolHash` freeze support for closed symbol sets. The freeze
  step builds a `hash -> symbol id` lookup and reports collisions explicitly
  before runtime payloads depend on hash-only dispatch.
- Added native variant layout allocation: the tag is stored as a raw `Symbol`
  field and typed payload fields remain in the same monomorphic payload buffer.
- Added `crates/yulang-native/src/mmtk_runtime.rs` and made MMTk a normal
  native dependency. The boundary configures MMTk options and starts with a
  single-threaded `NoGC` spike plan without changing the default native path.
- Added `crates/yulang-native/src/mmtk_binding.rs`. The
  skeleton implements the MMTk VM binding surface with a Yulang object header,
  `YValue` slot representation, memory slice, object-size callback, and
  trace-slot scanner. Unwired moving/collection paths are explicit prototype
  boundaries.
- Added the first `MmtkHeap` prototype. It implements `YHeap`, builds the
  single-threaded `NoGC` binding, allocates object headers and trace slots
  through MMTk, and stores the current semantic `YObject` payload in the MMTk
  raw payload area. `MmtkHeap::object` now projects the payload from the object
  reference instead of using a side table for semantic payloads.
- Split the `YHeap` read surface into object-header, trace-child, and full
  object projection. MMTk tracing and stats now read header/trace data from the
  MMTk object instead of requiring the full semantic payload.
- Added broader MMTk heap smoke coverage for debug rendering, allocation stats,
  root trace reports, closure/tuple/string reachability, rope nodes, temporary
  helper roots, and MMTk trace-slot scanning. MMTk-initializing tests are
  serialized because mmtk-core has process-global runtime state during the
  current prototype.
- Added `MmtkNativeRuntimeContext`, the first MMTk-backed native helper context
  over raw `YValue` words. It covers unit, bool, int arithmetic/compare,
  scalar string conversions, red-black-tree string
  construction/concat/index/range/splice/length/equality, compact bytes/path
  conversion helpers, compact tuple construction/access, compact record field
  update/select/default/has/without, compact variant construction/tag/payload
  access, chunked red-black list construction/length/index/view/range/splice,
  debug rendering, and a small C ABI roundtrip for future helper symbol wiring.
- Added a parallel `yulang_mmtk_cps_*` helper symbol set. The symbols use the
  MMTk helper context and pass tagged
  `YValue` words through i64 lanes. They deliberately do not replace the
  existing `yulang_cps_*` symbols yet, because the current CPS backend still
  lowers plain scalar i64 values and prototype heap handles.
- Added a minimal JIT smoke for the future all-`YValue` lane. The
  test builds a tiny Cranelift function by hand, registers the same
  `yulang_mmtk_cps_*` symbol set used by the CPS backend, and passes MMTk
  `YValue` strings, bytes, and tuples through JIT calls.
- Added `CpsReprCraneliftOptions::mmtk_yvalue_primitives()` as the first normal
  CPS repr lowering experiment for the MMTk lane. It routes string literals,
  the current runtime primitive family, and tuple/record/variant structural
  statements through `yulang_mmtk_cps_*` where that boundary already carries
  `YValue` payloads.
- Extended the opt-in MMTk CPS repr lane to raw string range/splice, the compact
  list subset, and tuple/record/variant structural statements. Structure
  payload lowering now boxes scalar literals into `YValue` at the construction
  boundary instead of mixing plain scalar i64 values into MMTk heap objects.
- Extended the same opt-in lane so Yulang `int`, bool, and unit values now flow
  as tagged `YValue` words through primitive calls and branch conditions. Float
  operations still use unboxed `f64` inputs and box boolean comparison results
  back into `YValue`. Fixed-width `i64`/`u64` lanes are still native layout
  support only, not source-level Yulang types.
- Added `yulang run --mmtk` as the benchmark entrypoint for the MMTk `YValue`
  primitive lane. The lane is intentionally fail-fast for unsupported shapes,
  matching the early native backend rollout style.
- Extended `MmtkHeap` so its object table distinguishes semantic `YObject`
  payloads, compact raw bytes, and compact native-block payloads. Header and
  trace-slot scanning work for all three; compact native blocks can read fields
  back through native layout offsets without projecting a full semantic object.
- Moved compact string/list rope node metadata onto raw payload bytes and taught
  the MMTk runtime to decode rope headers, colors, black heights, lengths, and
  child/value fields directly from object header/payload data.
- Added compact native-block allocation/read support for closure, thunk,
  resumption, continuation env, handler frame, and return frame payloads. The
  smoke test verifies code/prompt/continuation words, env slots, trace slots,
  and kind-specific stats without semantic `YObject` projection.
- Shaped string/list payloads as red-black-tree rope nodes aligned with the
  existing runtime `StringTree` / `ListTree` operations, so the GC object model
  does not bake in flat-only `str` / `list` storage.
- Kept native CPS handler, guard, and return-frame stacks on mutable vectors
  with cached snapshots. The first snapshot after mutation still clones the
  active stack. `Rc<Vec>` copy-on-write moved the cost to later mutation and
  regressed the nondeterminism benchmark; `im::Vector` removed full clones but
  had too much hot-path constant cost. A real fix needs a dedicated stack
  representation that keeps push/pop cheap while snapshots structurally share
  captured prefixes.
- Added compact MMTk `ControlStack` objects for the MMTk CPS lane. Captured
  native CPS snapshots now get a GC heap mirror whose trace slots include
  reachable MMTk `YValue` children while non-MMTk pointer-shaped words are
  filtered out. The filter keeps a tracked MMTk address range so prototype
  closure/continuation pointers can be rejected before the object-table lookup,
  and skips allocating snapshots that introduce no MMTk heap children.
- Added `mmtk_cps_control.rs`, a dedicated helper module that can allocate
  closure/thunk/resumption bodies as compact fixed raw-payload MMTk objects and
  read code/env slots directly. The control payload now carries its own env
  length, i64-env allocation avoids the extra `Vec<YValue>` materialization, and
  the helper context uses a raw thread-local context pointer instead of a
  `RefCell<Option<_>>` borrow on every helper call. Direct helper tests pass.
  The Cranelift `--mmtk` lane can opt into the path with
  `YULANG_MMTK_CPS_CONTROL_OBJECTS=unsafe`, but default lowering still uses the
  existing native closure/thunk control helpers. `YULANG_MMTK_CPS_CONTROL_OBJECTS=1`
  is kept on the stable path because the compact control body does not yet
  capture native handler/guard snapshots. After the raw context pointer and
  heap-range prefilter changes, the all-compact thunk experiment showed a
  list-only speedup but mis-evaluated `.once` shapes because boundary conversion
  recreated native thunks from the boundary-time context. Re-enabled compact
  thunk creation only when the native control capture context is empty, and made
  compact-thunk boundary conversion build the prototype thunk from an explicit
  empty snapshot. Compact closures can still be exercised, non-empty capture
  contexts still fall back to prototype helpers, and `showcase` / isolated
  `.once` tail both return `just 18`. Added a narrower helper-level sidecar
  experiment behind `YULANG_MMTK_CPS_CONTROL_THUNK_SIDECARS=unsafe`: it keeps a
  compact thunk surface while storing the creation-time native thunk snapshot in
  a side table, and direct helper tests cover force/boundary basics. The native
  force helper now delegates back to MMTk force when a mixed native path receives
  a compact thunk. Full lowering keeps the sidecar bridge disabled because the
  showcase path still exposes a non-local-flow hole through compact apply/force
  when the sidecar is forced broadly.
- Updated README/native docs/tasks/design note to keep the default native path
  clearly marked as prototype while the GC runtime object model evolves.
- Tightened the MMTk benchmark lane after the first slowdown pass:
  - compact list `len` and indexed access now read list-tree metadata instead
    of flattening the whole tree;
  - compact list `view_raw` now returns stored node children directly instead
    of splitting and rebuilding subtrees, and compact list flatten/debug paths
    read trace slots without per-node temporary `Vec` allocation;
  - compact tuple/variant values now use fixed raw-payload metadata and trace
    slots instead of compact native-block layout projection, so hot list-view
    tuple/variant construction no longer interns a native layout;
  - compact record values now use raw-payload field-name metadata plus trace
    slots for field values instead of compact native-block layout projection;
  - MMTk compact record/variant debug rendering no longer falls through to the
    semantic `YObject` projection, so MMTk roots such as `just (3, 4, 5)` render
    like the prototype native path;
  - native CPS control-stack `ControlStack` mirrors are now opt-in via
    `YULANG_MMTK_CPS_CONTROL_STACK_SNAPSHOTS=1`, because default `--mmtk`
    currently uses the `NoGC` benchmark lane and should not pay root-mirror
    cost before collection is wired.
- Release executable checks after the tightening:
  `(each 1..20 + each 1..20).list.len` is roughly 0.40-0.43s for both native
  and MMTk after dropping one native outlier, while `.list.say` is roughly
  native 0.62-0.68s vs default MMTk 0.65-0.71s on this machine. With thunks
  forced back to the native helper path, `YULANG_MMTK_CPS_CONTROL_OBJECTS=unsafe`
  was around 0.70s on the same `.list.say` executable. After re-enabling compact
  thunk creation with empty-snapshot boundary conversion, `examples/showcase.yu`
  sequential release runs were native 1.63-1.71s, default MMTk 1.59-1.91s, and
  unsafe compact-control 1.55-1.69s. The initial 1.8-1.9s MMTk slowdown is gone,
  but MMTk is not yet a stable general win over the prototype native effects
  lane.
- The remaining MMTk performance issue is structural: the compact control lane
  still falls back when non-empty handler/guard/blocked-effect snapshots must be
  captured, boundary thunks cross back through a prototype thunk, the sidecar
  bridge is not yet safe for full lowering, hot values are not fully unified on
  the compact `YValue` layout, and CPS capture still needs a dedicated stack
  that keeps push/pop cheap while snapshots share structure.
