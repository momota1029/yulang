# 2026-05-18 Native Release Prep

## Completed

- Fixed the remaining recursive handler family that blocked a native release
  candidate:
  - recursive tuple-returning handler no longer leaks thunk/heap handles as
    pointer-looking integers;
  - recursive handler block-tail loop control no longer collapses to the local
    `5` value before the value arm / tuple construction finishes;
  - unit and bool root display keep user-facing value shape on covered native
    paths.
- Moved the fixed native handler notes into `notes/bugs/solved/` and updated
  the bug indexes.
- Refactored the large native CPS Cranelift runtime helper block into
  `crates/yulang-native/src/cps_repr_cranelift/runtime_i64.rs`, leaving the
  backend entry file easier to navigate.
- Split effectful direct-call reification into
  `crates/yulang-native/src/cps_effectful_calls.rs` and run it before capture
  inference, so recursive handler calls get an explicit resume continuation.
- Updated `README.md`, `docs/native-backend.md`, `docs/status.md`,
  `notes/design/native-remaining-failure-matrix.md`, and `tasks/current.md`
  for release prep. The result is not a release candidate yet because N11/N13
  still fail the gate.

- Passed: `RUSTC_WRAPPER= cargo test -q -p yulang-native`
- Passed: `RUSTC_WRAPPER= cargo test -q -p yulang --test native_cli`
- Failed: `RUSTC_WRAPPER= cargo test -q -p yulang-compile runs_undet_once_open_range_guard_through_cps_repr`

- Failed: `RUSTC_WRAPPER= cargo test -q -p yulang-compile runs_junction_method_undet_once_through_cps_repr`
  - expected `:just 18`
  - CPS repr eval produced `CpsValueId(usize::MAX)` at the root plain-value boundary
- native CLI smoke:
  - matched the expected covered roots for `examples/03_for_last.yu`,
    `examples/04_sub_return.yu`, `examples/07_junction.yu`, and
    `examples/10_effect_handler.yu`

  - did not match VM for the final root of `examples/showcase.yu`

Conclusion: do not cut the native experimental release yet. Fix N11/N13 first.
