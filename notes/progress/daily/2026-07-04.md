# 2026-07-04

## Host suspend one-shot runtime slice

- Wired suspend one-shot host operations through the existing host ABI registry
  and runtime scheduler queue.
- `HostCtx::suspend_one_shot()` now issues an opaque resume token. The token can
  only enqueue a `BoundaryValue` resume event; it has no perform/eval/runtime
  reentry API.
- Runtime host dispatch now branches by manifest tier:
  - `sync` keeps the existing immediate `HostOutcome::Return` path.
  - `suspend-one-shot` parks the escaped request continuation and resumes it
    from a scheduler `Resume` event.
  - `suspend-multi-shot` remains a reserved tier and returns an explicit
    unimplemented runtime error.
- Added in-process fake host tests for normal suspend/resume, cancel-before-
  resume rejection, one-shot double-resume rejection, and multi-shot reservation.
- Validation:
  - `RUSTC_WRAPPER= cargo test -p evidence-vm -- --test-threads=1`
  - `RUSTC_WRAPPER= cargo check -q -p evidence-vm -p yulang`
  - `cargo fmt --all -- --check`

## Parser pattern Stage P2 rest slice

- Added rule-body `..` parsing for explicit `rule { ... }` and rule literal
  interpolation (`~"{..}"` / `~"{r = ..}"`). Bare `..` inside `~"..."` remains
  literal text.
- Added pure std parser `std::text::parse::rest`, implemented with
  `snapshot` + `advance input.len`, and lowered rule rest to that parser.
- Kept rest terminal: `..` after further rule items reports
  `yulang.rule-rest-position`.
- Changed parser-pattern capture binding from synthetic field selections to a
  record-pattern destructure of the parser result. This fixes capture guard dot
  method selection such as `id.starts_with "A"` without changing general method
  resolution.
- Extended `parser_pattern_case` with lazy capture, interpolation capture, bare
  parser reference capture, guard fallthrough, rest capture/discard, string
  interpolation rest capture, and dot-guard regression.
- Validation:

## Host suspend multi-shot runtime slice

- Added `HostCtx::suspend_multi_shot()` and multi-shot resume token handling.
- Multi-shot resume tokens can enqueue more than one resume. Each processed
  resume spawns a structured child branch under the branch that suspended the
  host operation, then runs a cloned continuation with the `BoundaryValue`
  payload.
- Added token `close()`. Close rejects later resumes and removes the runtime
  continuation template; it does not synthesize a final parent value.
- Parent cancel/drop drops the token and later resumes return `Dropped`.
  Child cancel stays on the existing cancel queue and does not affect sibling
  children or the parent token.
- Kept server/net public surface, manifest tier assignment, and std unchanged.
- Validation:
  - `RUSTC_WRAPPER= cargo test -p evidence-vm host_abi_suspend_multi_shot -- --test-threads=1`
  - `RUSTC_WRAPPER= cargo test -p evidence-vm scheduler -- --test-threads=1`
  - `RUSTC_WRAPPER= cargo test -p evidence-vm host_abi_suspend_multi_shot_parent_cancel_rejects_resume_and_cascades_children -- --test-threads=1`
  - `cargo fmt --all -- --check`
