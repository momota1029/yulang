## Evidence VM catch body env-preserving proof

- Added a static `env_preserving_exprs` table to `RuntimeEvidenceRunner`.
  - The table proves only this local property: evaluating an expression does not
    extend the `Env` value passed by its caller.
  - `Stmt::Let` and `Stmt::Module` remain non-preserving.
  - `Catch` and `RefSet` are preserving because they isolate their mutable body /
    reference work behind their own env clones.
  - `Apply` is preserving when the callee expression is preserving; non-immediate
    args are already evaluated in an internal cloned env.
- `eval_catch` now reuses the caller env for catch bodies whose table entry is
  preserving, and keeps the old cloned-body path otherwise.
- Added counters:
  - `catch_body_env_clone_elided`
  - `catch_body_env_clone_kept`

  - `file::open` / `open_with` as file session entrypoints.
  - `.meta`, `.text`, `.bytes`, and `.raw` as session views.
