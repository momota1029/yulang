# 2026-06-12

## yulang2 dump

- `dump-poly-std-in <path> <module>` を追加した。
  - std 込みで全体を lower / infer してから、指定 module 直下の値だけを compact poly dump する。
  - module summary に values / types / child modules / typed lets / missing schemes / bodyless declarations /
    dirty stack schemes / local lowering errors を出す。
  - stderr へ出す body lowering error は、指定 module 直下の値に紐づくものだけへ絞る。
- `dump-poly-std-in-raw <path> <module>` を追加した。
  - 指定 module の root value に加え、関数引数や局所 let など、dump 対象本体内の local def を raw
    `defs` に補う。
- `infer::ModuleTable` に、module 直下の value / type / child module decl を宣言順で返す API を追加した。
- `poly::dump` に root def を指定して compact/raw dump する入口を追加した。

## yulang2 check

- `check-poly-std <path>` を追加した。
  - dump を作らず、std 込みの collect / load / infer / summary timing を出す。
  - module ごとの direct value / typed / missing scheme / bodyless declaration / stack scheme / local error を出す。
  - arena 内の全 `Def::Let` について typed / missing scheme / bodyless declaration を集計し、未型付け def を列挙する。
  - `TypeMismatch` / unresolved type name / unresolved value name を短い diagnostic として表示する。
- `check-poly-std-in <path> <module>` を追加した。
  - 全体推論は一回だけ走らせ、指定 module の summary / missing schemes / bodyless declarations /
    local diagnostics だけを表示する。
- `file_handle` を builtin 型として追加した。
  - `std::io::file` の act operation signature が解けるようになり、`file_handle` unresolved error が消えた。
- `examples/showcase.yu` では、手元実行で概ね次の内訳だった。
  - collect: 約 232ms
  - load: 約 238ms
  - infer: 約 1.02s
  - summarize: 約 2ms
  - total: 約 1.49s
  - missing schemes: 13
  - bodyless declarations: 138
  - lowering errors: 0
