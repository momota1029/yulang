- `specialize` が `Scheme.stack_quantifiers` を unsupported として拒否していた制限を外した。
- scheme matching では `Stack(inner, weight)` の `inner` を expected type に直接合わせないようにした。

  - `all_except(signal)` が `[signal]` を消し、root 側に余計な `force-thunk[thunk[stack(...), _]]` が残らないことを確認した。

## yulang specialize: polymorphic stack handler call

- `pub act sub 'a` の `sub::sub: sub::return 0` が `dump-mono` まで通るようにした。
- `Neu::Bounds(lower, upper)` の materialize で lower だけを採用して upper 側の関係を捨てていたため、effect family argument 経由で `'a = int` が伝わっていなかった。
  - lower/upper の両方がある場合は intersection として保持し、`T & T` を `T` に簡約する。
- effect を含まない具体値にかかった `stack(T, weight)` は `T` に簡約する。

## sub syntax and labelled return

- `sub` を parser-level syntax にした。
  - `sub:` / `sub 'label:` は `Expr > SubExpr` として出る。
  - `\sub ... -> ...` / `\sub 'label ... -> ...` は `Expr > SubLambdaExpr` として出る。
  - optional label は `SubLabel` node にした。
  - `sub` は contextual keyword 扱いで、`sub::sub` や `act sub` は通常の name のまま残す。
- `sub:` / `sub 'label:` の lowering を現行 pipeline へ入れた。
  - `sub:` は `std::control::flow::sub::sub` へ desugar し、body 内の裸 `return` を
    `std::control::flow::sub::return` へ解決する。
  - `sub 'label:` は syntax site ごとに `std::control::flow::label_sub` の synthetic act copy
    を作り、body 内の裸 `return` と `'label.return` をその copy の `return` operation へ解決する。
  - `\sub` は通常 lambda の parameter / annotation lowering を使い、最終 body だけを
    `sub:` / `sub 'label:` と同じ control scope で lower する。
  - synthetic act copy の operation effect type vars は、copy 先に `act_template` がない場合でも
    resolved copy 元から復元するようにした。これにより `return: 'a -> [label_sub 'a] never`
    の `'a` が `any` に落ちない。
- root expression 直下で synthetic act copy が必要な場合に備えて、pass1 で root expr owner を
  必要な root にだけ採番し、pass2 が同じ owner を消費する table を足した。
- `lib/std/control/flow.yu` の `label_sub` は copy 先でも自己 act を参照できるよう、
  `control_label` の constructor と catch arm の `return` を bare 参照にした。
- root expression 直下の labelled `sub` で、synthetic copy 内の `control_label` が
  runtime root として漏れる問題を直した。
  - synthetic act copy body の computed binding は fetch 情報だけ残し、source-level runtime root
    にはしない。
  - synthetic owner を持つ root expression だけ hidden root def を持たせ、infer で閉じた root
    boundary scheme を specialize の expected type として渡す。
  - root expression lowering でも binding と同じく child level を切り、hidden root scheme に
    root-level free var が漏れないようにした。
- regression:
  - `run_control_source_text_with_embedded_std_lowers_sub_syntax_return`

## nested labelled sub hygiene and specialize

- nested labelled `sub` lambda と root labelled `sub` の組み合わせを直した。
  - `\sub 'inner x -> 'inner.return x` は、外側 handler の marker が inner handler を飛ばさず
    inner handler 自身で捕まえる。
  - `\sub 'inner x -> 'outer.return x` は、inner handler を素通りして外側 labelled `sub` に届く。
- runtime guard marker に `handler_frame` を持たせた。
  - MarkerFrame と、値へ染みた marker / function adapter marker を区別する。
  - request の guard 判定は、operation path に対応する最初の active handler frame より内側で
    push された guard id だけを見る。
  - これにより同じ `sub` path の callback hygiene は維持しつつ、別 synthetic label path の
    inner handler は自分の effect を処理できる。
- specialize の block `let` RHS は、単純 var pattern なら poly の principal scheme を expected type
  として使うようにした。
  - 期待型なしで RHS を再推論すると、discard される local function return に scheme 内の未使用

## optional records and poly variants

- record pattern default を `poly` / `mono` / `control-vm` の pattern IR に残すようにした。
  - `RecordPatField { name, pat, default }` を追加し、lowering で default expr を捨てない。
  - specialize は record pattern default を field 型の文脈で解き、missing optional field の concrete
    slot に default の型が入るようにした。
  - mono-runtime / control-vm は missing field の default を pattern binding 時に評価してから nested
    pattern に渡す。
  - default 評価は pattern bind 専用の CPS に乗せた。default 内で effect request が発生しても
    outer `catch` に届き、resume 後は残りの pattern bind へ戻れる。
- empty brace expression `{}` は empty record literal として lower するようにした。
  - `my f({x = 1}) = x; f {}` が unit argument ではなく `{}` argument になる。
- specialize の concrete slot join/meet に record / poly variant の構造規則を足した。
  - record lower-bound join は片側にだけある field を optional 化する。
  - poly variant lower-bound join は tag + arity ごとに variant set を union する。
  - upper-bound meet は record field を要求側へ寄せ、poly variant は共通 tag + arity の交差へ寄せる。
- runtime boundary equivalence は optional record widening と poly variant widening を no-op として扱う。
