# 2026-05-15 Native Progress

## Completed

- CPS repr Cranelift now identifies thunk / closure pointer handles in display
  instead of leaking anonymous pointer-looking integers.
- Recursive handler tuple results now match VM on the forced/default CPS repr
  executable path. The fix had two parts:
  - `force_thunk_i64` updates an active `ScopeReturn` payload when it forces
    that exact thunk handle.
  - selected handler env lookup reads duplicate entries newest-first.
- Added `runs_recursive_effect_handler_tuple_result_through_cps_repr`.
- Forced value-backend execution rejects top-level structural pattern binding
  shapes instead of emitting a crashing executable. Default native already
  routes these roots to CPS repr.
- Re-ran the passing example sweep; `examples/10_effect_handler.yu` now passes
  on default native.

## Still Open

- `examples/03_for_last.yu` / open-range local `last` still times out on CPS
  repr Cranelift.
- Filtered trace shows the local `last` handler is selected and routed, but the
  range/fold recursion continues installing fresh handler frames afterwards.
- The interpreter handles a current-handler `ScopeReturn` by mutating the eval
  loop continuation (`current = target; args = [value]`) rather than calling the
  escape continuation as a nested host function. Native still uses the latter,
  so N9 likely needs either a native eval-loop jump analogue or scoped
  short-circuit consumption at the correct continuation boundary.
