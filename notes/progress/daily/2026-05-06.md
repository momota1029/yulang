  - infer lower
  - type rendering
  - diagnostics

  - runtime lower
  - monomorphize

  result pane.
- This is not yet a cached infer artifact. It is the safe first cut before
  splitting def ids, type vars, role tables, and principal evidence across a

- Added source origins and process-local lowered std caching:
  - `SourceOrigin::{Entry, Std, User}` separates std from user source files.
  - Wasm stores a lowered std `LowerState` in `SourceLowerCache`.
  - `RunOutput.timings` reports source cache hits/misses/build/clone time.
  - Playground warms the full bundled std cache after the first run so later
    runs hit the cache without blocking first paint or initial execution.
- Added the first `StdInferSnapshot` API:
  - `SourceLowerCache` stores snapshots rather than raw lowered states.
  - `build_std_infer_snapshot` builds the current transitional snapshot.
  - `lower_source_set_with_std_snapshot` lowers entry/user code from an explicit
    snapshot and has a regression test against full uncached lowering.
- Added the first serde boundary for persistent std snapshots:
  - `StdInferSnapshotManifest` stores a snapshot format version and source hash
    key.
  - The manifest round-trips through `serde_json` in the existing std snapshot
    regression test.
  - This intentionally serializes only the cache identity for now, not the full
    lowered state.
- Added `StdInferSnapshotData` as the first persistent payload shell:
  - includes the manifest plus stable path/def indexes for value symbols, type
    symbols, and effect operations.
  - round-trips through `serde_json` alongside the process-local snapshot test.
  - still does not instantiate user lowering from bytes; the old cloned state
    remains the behavioral oracle.
- Exposed the payload shell through wasm as `std_snapshot_data()` for build-time
  tooling experiments. Snapshot symbol ids are path-order local ids rather than
  raw `DefId` values, because raw ids depend on global allocation order and are
  not stable across repeated builds.
- Extended `StdInferSnapshotData` with a serializable module/name table:
  - module ids are path-order local ids rather than raw `ModuleId` values.
  - value/type/operator/module entries point at snapshot-local symbol/module ids.
  - this is still payload-only; no byte-loaded instantiation path exists yet.
- Added validation for the serialized snapshot payload so unsupported format
  versions and broken local id references are rejected before a future import
  path tries to instantiate them.
- Added first scheme / role / effect metadata payload:
  - value schemes are keyed by snapshot-local value symbol ids and rendered from

  - role arg info, role methods, role impl members, effect declarations, and

## Principal Elaborate Hot Path

- Added `principal_elaborate` timing buckets inside the monomorphize pass
  profile. The first useful split showed:
  - `rewrite-root-exprs` dominates `examples/showcase.yu`
  - `intern-rewrite-body` is the largest named sub-cost
  - plan construction/completion is small by comparison
- Removed a redundant full-subtree `project_runtime_expr_types` call from the
  `Apply` rewrite path after callee/arg subexpressions have already been
  rewritten. This avoids rewalking the apply subtree for every nested apply.
- Removed a clone-heavy role-method precheck at the start of the `Apply` rewrite
  path. It built an apply spine from cloned callee/arg subtrees before the
  normal owned rewrite path, causing repeated subtree cloning on nested calls.
  Role/impl rewrites now go through the normal post-rewrite apply path.
- Collapsed the apply readiness check into the main principal rewrite attempt
  so the apply spine is built once per visited apply. Non-target calls now
  return quietly instead of polluting skip reachability with already specialized
  or ordinary non-generic callees.
- Local release timing for `examples/showcase.yu` improved roughly:
  - principal-elaborate pass: about 240-250ms -> about 150-160ms
  - total monomorphize: about 250ms -> about 160ms
- `examples/07_junction.yu` strict principal elaboration still passes and is
  around 10-14ms for principal-elaborate in local release timing.
- The next hot area is still root/body rewrite traversal, not principal plan
  solving. Avoid returning to graph solving unless a surviving actionable
  target appears.

## Persistent Std Snapshot Import

- Added the first conservative import API for persisted std snapshot payloads:
  - `import_std_infer_snapshot_data`
  - `StdInferSnapshotImport`
  - structured missing path reports for modules, values, and types
- The importer validates the payload, installs builtin primitives into a fresh
  `LowerState`, and resolves snapshot-local module/value/type ids back to the
  defs/modules that already exist in that builtin state.
- Metadata that references snapshot-local ids is also projected through those
  import tables:
  - scheme -> value def
  - role method -> value def
  - role impl member -> value def
  - effect method -> value def and module id
- Import now restores the module skeleton from snapshot module paths before
  resolving symbol references. This is still not a fake value/type import: it
  only recreates the namespace container so later real body/scheme import has a
  place to attach.
- The import result now includes coverage counters for resolved module, value,
  type, scheme, role-method, and effect-method references. This gives wasm and
  CLI callers a cheap way to decide whether a persisted snapshot is only a
  partial index or complete enough to use as a future std lower replacement.
- wasm now exposes `std_snapshot_import_coverage()` so the playground can inspect
  the current persisted snapshot restoration coverage without running it as the
  main compiler path yet.
- Added explicit coverage predicates:
  - `can_replace_std_lowering()`
  - `has_partial_value_or_type_import()`
  These make the current safety line visible: partial imports can be inspected,
  but they must not replace std lowering until value/type/scheme and role/effect
  references are all restored.
- Made `yulang-core-ir` serde-compatible and added a JSON roundtrip test for
  `CoreProgram`. This is the first direct step toward persisting the std core
  artifact instead of trying to serialize the full infer/lower state.
- Added `StdCoreSnapshotData` and `build_std_core_snapshot_data`.
  This builds std from sources, finishes inference, exports all collected std
  bindings into a serializable `CoreProgram`, and keeps the same std snapshot
  manifest key.
- wasm exposes `std_core_snapshot_data()` for inspection. The real std snapshot
  includes source-defined bindings such as `std::list::fold_impl`, so this is
  now a plausible persistent artifact for bypassing std infer/export later.

## Compiled Unit Cache Correction

- Corrected the cache direction after reviewing the std snapshot prototype:
  std-only and core-only artifacts are not enough.
- Wrote the detailed plan in
  `notes/design/partial-compilation-cache-plan.md`.
- Extended the plan with the operator split:
  - syntax surface carries cached parser operator definitions and reexports;
  - namespace surface carries `(operator name, fixity) -> value symbol`
    identity.
- Started Phase 1 of the compiled-unit cache:
  - added `CompiledUnitManifest`;
  - added `CompiledSyntaxSurface`;
  - added stable `CompiledOperatorSyntax` instead of serializing parser
    `OpDef` directly;
  - added JSON roundtrip and operator-table reconstruction tests.
- Extended syntax artifacts with module-level syntax surfaces and an import
  helper that rebuilds a downstream `OpTable` from cached dependency artifacts.
- Started Phase 2 with a read-only `CompiledNamespaceSurface` builder from
  `LowerState` + source units:
  - module-level value/type/operator namespace entries;
  - stable unit-local value/type ids;
  - operator value identity as `(name, fixity) -> unit value symbol`;
  - JSON roundtrip coverage in tests.
- Added namespace-surface validation so broken module value/type/operator symbol
  references are detected before a future import path creates process-local
  ids.
- Added a first namespace import skeleton:
  - validates namespace artifacts;
  - allocates fresh process-local `DefId` / `TypeVar` ids;
  - restores module skeletons, values, types, and operator value entries;
  - merges multiple unit surfaces while remapping unit-local symbol ids.
- Started Phase 3 with a read-only `CompiledTypedSurface`:
  - schemes;
  - roles / role methods / role impl member refs;
  - effects / effect methods;
  - validation that typed metadata only refers to namespace value symbols in
    the same artifact.
- Yulang source files export operator syntax, and downstream parsing depends on
  those operators before inference/export. A reusable cache must therefore carry
  syntax exports/operator-table contributions along with typed artifacts.
- Added `SourceSet::compilation_units()` in `yulang-source`.
  It exposes:
  - file dependency SCCs
  - unit dependency edges
  - unit origin classification
  - syntax exports collected per unit
- This sets the next cache boundary to file SCCs:
  `SourceCompilationUnit -> CompiledUnit artifact`, rather than a special-case
  std snapshot.
- `StdCoreSnapshotData` remains useful as a serialization experiment for the
  runtime-facing part of a unit, but it should not be treated as the final
  persistent cache format.
- It deliberately does not create fake defs for source-defined std entries yet.
  Missing source std modules/values/types are kept as structured import gaps so
  the next restoration step can target real scheme/body import rather than
  pretending the state is complete.
- Added a regression test showing that a builtin symbol such as
  `std::int::add` resolves during partial import, while a source-defined std
  module remains reported as missing.
