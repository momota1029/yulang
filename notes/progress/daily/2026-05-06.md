## Partial Compilation

- Added `RunOutput.timings` for Wasm runs:
  - source set
  - infer lower
  - type rendering
  - diagnostics
  - core export
  - runtime lower
  - monomorphize
  - VM compile/eval
- The first playground-side std `SourceFile` cache attempt was reverted after
  measurement. Source loading was not the dominant cost, and the cache path
  added more complexity than speed.
- Playground timing output now goes to `console.debug` instead of the visible
  result pane.
- This is not yet a cached infer artifact. It is the safe first cut before
  splitting def ids, type vars, role tables, and principal evidence across a
  real partial-compilation boundary.
- Added source origins and process-local lowered std caching:
  - `SourceOrigin::{Entry, Std, User}` separates std from user source files.
  - Wasm stores a lowered std `LowerState` in `SourceLowerCache`.
  - `RunOutput.timings` reports source cache hits/misses/build/clone time.
  - Playground warms the full bundled std cache after the first run so later
    runs hit the cache without blocking first paint or initial execution.
- Added the first `StdInferSnapshot` API:
  - `SourceLowerCache` stores snapshots rather than raw lowered states.
  - `build_std_infer_snapshot` builds the current transitional snapshot.
  - `lower_source_set_with_std_snapshot` lowers entry/user code from an explicit
    snapshot and has a regression test against full uncached lowering.
- Added the first serde boundary for persistent std snapshots:
  - `StdInferSnapshotManifest` stores a snapshot format version and source hash
    key.
  - The manifest round-trips through `serde_json` in the existing std snapshot
    regression test.
  - This intentionally serializes only the cache identity for now, not the full
    lowered state.
- Added `StdInferSnapshotData` as the first persistent payload shell:
  - includes the manifest plus stable path/def indexes for value symbols, type
    symbols, and effect operations.
  - round-trips through `serde_json` alongside the process-local snapshot test.
  - still does not instantiate user lowering from bytes; the old cloned state
    remains the behavioral oracle.
- Exposed the payload shell through wasm as `std_snapshot_data()` for build-time
  tooling experiments. Snapshot symbol ids are path-order local ids rather than
  raw `DefId` values, because raw ids depend on global allocation order and are
  not stable across repeated builds.
- Extended `StdInferSnapshotData` with a serializable module/name table:
  - module ids are path-order local ids rather than raw `ModuleId` values.
  - value/type/operator/module entries point at snapshot-local symbol/module ids.
  - this is still payload-only; no byte-loaded instantiation path exists yet.
- Added validation for the serialized snapshot payload so unsupported format
  versions and broken local id references are rejected before a future import
  path tries to instantiate them.
- Added first scheme / role / effect metadata payload:
  - value schemes are keyed by snapshot-local value symbol ids and rendered from
    stored compact schemes or, for primitive/builtin entries, from `DefId ->
    TypeVar` fallback compaction in the artifact builder path.
  - role arg info, role methods, role impl members, effect declarations, and
    effect methods are serialized with snapshot-local references.
  - this still keeps runtime behavior on the process-local `LowerState` clone;
    the payload is now rich enough to start designing an import path.
- Updated `notes/todo.md` and `tasks/current.md` to park whole-`LowerState`
  cloning as a stepping stone and make persistent `StdInferSnapshot` the next
  partial-compilation target.

## Principal Elaborate Hot Path

- Added `principal_elaborate` timing buckets inside the monomorphize pass
  profile. The first useful split showed:
  - `rewrite-root-exprs` dominates `examples/showcase.yu`
  - `intern-rewrite-body` is the largest named sub-cost
  - plan construction/completion is small by comparison
- Removed a redundant full-subtree `project_runtime_expr_types` call from the
  `Apply` rewrite path after callee/arg subexpressions have already been
  rewritten. This avoids rewalking the apply subtree for every nested apply.
- Removed a clone-heavy role-method precheck at the start of the `Apply` rewrite
  path. It built an apply spine from cloned callee/arg subtrees before the
  normal owned rewrite path, causing repeated subtree cloning on nested calls.
  Role/impl rewrites now go through the normal post-rewrite apply path.
- Collapsed the apply readiness check into the main principal rewrite attempt
  so the apply spine is built once per visited apply. Non-target calls now
  return quietly instead of polluting skip reachability with already specialized
  or ordinary non-generic callees.
- Local release timing for `examples/showcase.yu` improved roughly:
  - principal-elaborate pass: about 240-250ms -> about 150-160ms
  - total monomorphize: about 250ms -> about 160ms
- `examples/07_junction.yu` strict principal elaboration still passes and is
  around 10-14ms for principal-elaborate in local release timing.
- The next hot area is still root/body rewrite traversal, not principal plan
  solving. Avoid returning to graph solving unless a surviving actionable
  target appears.
