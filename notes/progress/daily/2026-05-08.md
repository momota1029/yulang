# 2026-05-08

## std / path special case cleanup

- runtime monomorphize の `AssociatedSignatureKind` と associated demand signature closure を撤去した。
  - `for_in` / `Fold` / `Fold::find` / list view / handler collection / local var ref/run の target 分類で signature hole を補完しない。
  - known target から consumed effect を足す処理も外し、pure handler body から直接読める consumed effect だけを見る。
- complete principal export の role requirement 補完を削除した。
  - `item` associated type を `list` / `range` の path 末尾名から逆算しない。
  - substitution は call slot / exported bounds / expected evidence から取れる情報に限定する。
- runtime effect compatibility の `std::flow::loop` 特例を削除した。
  - expected effect に `std::flow::loop` が含まれても任意の actual effect は通さない。
  - parent effect と child operation の prefix match は通常規則として残す。
- numeric widening / union join / VM cast の末尾名 `int` / `float` 判定を削除した。
  - context を持たない層では bare `int` -> bare `float` の bootstrap fallback だけ残す。
- monomorphize type normalization の末尾名 `sub` payload 補完を削除した。
- VM effect operation erase の namespace 末尾名 fallback を削除した。
- ordinary role method call resolver の `cast` method 名分岐を削除した。
- ref list index projection を compiler export から外し、`lib/std/list.yu` の
  `Index (ref 'e (list 'a)) int` impl へ移した。
  - `&xs[i] = v` は std の index selection で child ref を作り、compiler は通常の
    ref set export だけを通す。
- runtime symbol export が root binding でない effect operation も出すようにした。
  - std/list から参照された `std::var::ref_update::update` が runtime lower で未束縛に
    ならないようにするための一般修正。
