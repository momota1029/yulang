## Positive effect row correction

- positive effect row の表現方針を確認した。
  - positive row 自体は持ってよい。
  - ただし negative row のような `items; tail` 表現ではない。
  - positive row は concrete effect atom と row variable を同列の item 列として持つ。
  - surface 表示は `[io, nondet, 'e, 'f]` のように comma 区切りで、`;` は使わない。
- `poly::types::Pos::Row(Vec<PosId>)` を追加した。
  - `Pos::Row` の items には `Pos::Con(...)` と `Pos::Var(...)` をそのまま並べる。
  - `poly::dump::type_format` は positive row を `"'[nondet, 'a]"` のように表示する。
- まだ annotation constraint 接続は未実装。
  - 次に確認する点は、`non-subtract([ζ] η, #{#c})` を現行の
    `Pos::NonSubtract(PosId, SubtractId)` / scheme predicate へどの段階で載せるか。

確認:

- `cargo test -q -p poly`

## Annotation constraint first cut

- pass2 annotation parser の出力 `AnnType` から `Pos` / `Neg` / `Neu` と effect subtract fact を作る `AnnConstraintLowerer` を追加した。
  - 同じ annotation scope の型変数は `AnnTypeVarId` ごとに solver `TypeVar` を共有する。
  - effect row wildcard `_` は具体型名とは分けて `AnnEffectAtom::Wildcard` として読む。
  - function return effect annotation は effect row bounds / subtract fact を作り、返り computation の `ret_eff` と `ret` を `Pos::NonSubtract(_, #id)` で包む。
  - computation 接続時には、同じ `#id` を target value/effect の `NonSubtract` self-edge にも載せる。これが current representation での predicate 側の接続点。
- `Pos::NonSubtract(pos, #id)` の constraint 伝播を追加した。
  - subtype edge に入ったら `#id` を left weight に足して `pos` へ進める。
  - 別の non-subtract table は増やしていない。
- lowering から annotation constraints を接続した。
  - binding header annotation は body computation へ接続する。
  - lambda parameter annotation は parameter computation へ接続し、そこで作った subtract id を lambda output computation にも載せる。

確認:

- `cargo test -q -p infer -- --nocapture`
- `cargo test -q -p infer -p poly -p sources -p yulang2`

## Effect annotation proxy update

- `spec/2026-05-31-effect-variable-subtractable.md` の更新へ追随した。
  - function arg effect annotation は exact row ではなく row lower だけを登録する。
  - function return effect annotation は fresh proxy effect var を作り、その proxy に subtract fact を付ける。
  - `[handled; 'e]` のように tail がある return effect では、tail を proxy の lower に入れつつ、subtract fact は tail ではなく proxy 側に置く。
  - computation effect annotation は target effect に subtractability を付け、row exact upper は作らない。

確認:

- `cargo test -q -p infer annotation::tests:: -- --nocapture`
- `cargo test -q -p infer -p poly -p sources -p yulang2`

## Ordinary expression subtract first cut

- 普通の application から、未注釈 local 引数 callee の return effect 制約を作る first cut を追加した。
  - `f x` の callee が lambda-local `Def::Arg` で、parameter annotation が無い場合だけ対象にする。
  - call の return effect slot に fresh `#a` で `Empty-subtract(γ, #a)` を登録する。
  - `#a` は現在の定義 lambda frame に記録し、その lambda の `ret_eff` と `ret` を
    `Pos::NonSubtract(_, #a)` で包む。
  - module/global callee には付けない。
  - anonymous lambda 内の application は外側の定義 lambda predicate へ上げない。
- `ExprLowerer::lower_binding_body_expr` を追加した。
  - 通常の `lower_expr` は anonymous lambda として扱う。
  - binding body 直下の lambda だけを、定義された関数の predicate を持つ lambda として扱う。
- 未注釈 parameter の function `arg_eff` は fresh var ではなく `Neg::Bot` にした。
  - 普通の関数は first cut でも、内部的に parameter effect が `never` だと分かる形へ寄せる。
  - `x: int` のような value-only annotation でも outer parameter effect は `never` のままにする。
  - `[handled] 'a` のような top-level effectful annotation だけ fresh effect slot を function `arg_eff` に出す。
- `poly::types::Neg::Bot` を追加した。
  - 単体表示は `never`。
  - function effect slot では省略する。
  - `a [never] -> [b] c` は `a -> [b] c`、`a -> [never] c` は `a -> c` と表示する。
- pure function subtyping の effect passthrough 規則を constraint 分解へ追加した。
  - `a -> [b] c <: d [e] -> [f] g` では、parameter effect を `e <: never` と比較しない。
  - 代わりに `e <: f` を登録し、純粋関数を通る引数側の効果を返り効果へ素通りさせる。
  - `e` と `f` はどちらも元の関数制約の upper 側にあるので、外側の right subtract weight を
    passthrough 制約の左右両方に載せる。

確認:

- `cargo test -q -p infer lowering::tests:: -- --nocapture`
- `cargo test -q -p infer -- --nocapture`
