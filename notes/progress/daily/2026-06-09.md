## Case / catch lowering first cut

- 新 infer の `ExprLowerer` に `CaseExpr` / `CatchExpr` lowering を追加した。
  - `case` は scrutinee の効果を結果効果へ流し、arm pattern と scrutinee value を双方向に接続する。
  - 各 case branch body の value / effect は結果 value / effect へ流す。
  - `case` / `catch` は構文的な computation として扱い、pure に見えても value 扱いにはしない。
- match/catch arm 用の pattern lowering を追加した。
  - bare ident は local binding のままにする。
  - path 付き、または payload 付き pattern は constructor spine として `Pat::Con` に落とす。
  - constructor pattern の `RefId` は `ModuleTable::value_path_at` で path 解決し、通常の ref use と同じ SCC 経路へ乗せる。
- `catch` の value arm / effect arm を lowering した。
  - value arm は scrutinee value と pattern value を接続し、branch body を結果へ流す。
  - effect arm は first pattern の operation path から effect family path を取り出し、その family を handled row item として読む。
  - continuation local には `payload [arg_eff] -> [scrutinee_eff] scrutinee_value` の上下界を入れる。
  - act body の binding 名を operation 一覧として pass1 で集め、catch の handled operation が完全なら `γ <: δ`、欠けがあれば `α <: δ` を入れる。
  - payload pattern が `_` / bare binding / 全要素 cover する tuple なら、その operation は完全に cover されたものとして扱う。
- guard / case lambda / catch lambda / indented block body はまだ未対応。

## Compact collect / simplify first cut

- `infer::compact` に `ConstraintMachine` の bounds table から `CompactRoot` を作る collect を追加した。
  - root `TypeVar` は positive side の `CompactType` として collect する。
  - lower bound の `weights.left` は covariant な `CompactVar` へ押し込む。
  - upper bound の `weights.right` は、upper 内で covariant になる位置だけに押し込む。
  - concrete `Con` / `Fun` / `Record` / `Tuple` / row shape 自体には weight を持たせない。
  - direct bound の bare variable はそこで止め、さらに上下界を展開しない。
  - structured bound 内の row tail などに出る変数は通常どおり展開する。
- positive effect row の variable item は `CompactRow.items` に閉じ込めず、同じ `CompactType.vars` へ出す。
  - `[α, γ, io]` のような row で、`α` / `γ` が同じ covariant group として共起分析に見えるようにする。
- 極性消去の first cut を追加した。
  - compact 全体で positive/negative の両方に出ない変数 occurrence を落とす。
  - `CompactBounds::Interval` の中心変数は positive/negative の双方に出るものとして扱う。
- 共起分析の first cut を追加した。
  - 同じ polarity で常に同じ flat group に出る変数を最小 `TypeVar` へ畳む。
  - weight は変数 occurrence に残し、代表変数への置換だけを行う。
  - shape exact key / sandwich lift / finalize での `S-subtract` 解釈はまだ未接続。
- 共起分析を weight-aware にした。
  - 同じ flat group に出ても、`CompactVar` の weight が違う変数は畳まない。
  - group key は `(group, weight)` として記録する。
- `sandwich_compact_root` を追加した。
  - 変数が主型全体で常に同じ単一 concrete kind と共起していれば、ネストした `CompactBounds::Interval` を `Con` / `Tuple` / `Fun` へ lift する。
  - `list` のような named constructor は `CompactBounds::Con` に畳む。
  - lift で新しい `Interval` を作るとき、lower / upper の両側に同じ `TypeVar` があればそのうち最小 ID を中心に使う。
  - 共通変数が無い場合は compact root 内の最大 `TypeVar` の次から fresh center を作る。
  - root の `CompactType` 自体はまだ final `Pos` へ戻していない。

## Compact quantification first cut

- `compact.rs` が大きくなりすぎていたため、極性消去・共起分析・sandwich・compact var rewrite を
  `infer::compact::analysis` に分けた。
  - 型定義 / collect / merge / finalize は `compact.rs` に残す。
  - 外向き entrypoint は re-export し、呼び出し側の API は変えない。
- `infer::generalize` module を追加した。
  - `generalize_type_var` は `compact_type_var -> simplify_compact_root -> generalize_compact_root`
    の pipeline を持つ。
  - `generalize_compact_root` は、すでに簡約済みの `CompactRoot` に量化集合を付ける入口。
  - `GeneralizedCompactRoot` は `CompactRoot` を保持し、finalize 後も compact 代入・subtract pruning に
    使える情報を捨てない。
- 量化規則は `generalize(lvl)` として、`level > lvl` の自由変数だけを quantifier にする。
  - `level <= lvl` の外側変数は non-generic として扱う。
  - 追加の `non_generic` set を引数に取り、値制限や外部露出変数を後で差し込めるようにした。
- `generalize_type_var` が呼ぶ compact simplify は `boundary.child()` を使う。
  - 量化候補と同じ `level > boundary` の変数だけが極性消去・共起統合の対象になる。
  - collect 展開は引き続き level 非依存。
- `CompactBounds::Interval.self_var` と `CompactRoot.rec_vars` も free var として見る。
  - recursive side table は `GeneralizedCompactRoot.compact.rec_vars` として残し、`poly::Scheme` へ無理に埋め込まない。
- quantified var に登録された subtract fact は、量化集合が決まった後の candidate subtract として
  `GeneralizedCompactRoot.subtracts` に ID だけ保持する。
  - subtractability 本体はまだ `poly::Scheme` に入れる型がないため、次の設計対象。
- `finalize_generalized_compact_root` は出口としてだけ残した。
  - `poly::Scheme` へ戻す時に `finalize_compact_root` を呼ぶ。
  - 通常の量化結果は compact 表現を保持する。

  - 量化変数に登録された subtract fact を candidate として集める。

  - すべての共変 occurrence がその id で塞がれている candidate は消し、対応する id の weight も

  - `target: Some(var)` は共起分析などで `source` を `var` へ寄せたことを表す。
  - `target: None` は極性消去で `source` の occurrence が消えたことを表す。
  - `simplify_compact_root` は各段の rewrite を集め、`a -> b, b -> None` のような連鎖を最終 target へ畳んで返す。

## Effect row upper bound subtraction

- `spec/2026-05-31-effect-variable-subtractable.md` の `α #A <: #B [S; β]` を `ConstraintMachine` に入れた。
  - `Pos::Var(α) <: Neg::Row(S, β)` で tail が `Neg::Var(β)` のときだけ専用処理する。
  - `α` の `S-subtract(α,#a)` fact のうち、left weight `#A` に含まれないものを active fact として集める。
  - row items `S` を active fact の subtractability と順に交差する。
    - `All` はそのまま。
    - `Empty` は空。
    - `Set(path)` は同じ effect family だけ残す。
    - `AllExcept(path)` はその family を除く。
  - active fact は tail `β` にも登録する。
  - 交差後の row items が空なら、`α` の upper bound には row 全体ではなく tail `β` だけを登録する。
  - 空でなければ、`[U; β]` を `α` の upper bound に登録する。
- tail が変数ではない `Neg::Row` は、現時点では ordinary upper bound として残す。
  - `[]` / `Top` tail のような既存の pure effect upper を壊さないため。
  - spec の対象は `α #A <: #B [S; β]` なので、tail 変数が見えるところから実装した。
- `Pos::Row <: Neg::Row` の経路でも同じ規則を使うことを固定した。
  - concrete effect item は matching upper item があれば effect 引数を不変比較する。
  - matching upper item がなければ upper tail へ流す。

## Ordinary expression subtract first cut

- 普通の application から、未注釈 local 引数 callee の return effect 制約を作る first cut を追加した。
  - `f x` の callee が lambda-local `Def::Arg` で、parameter annotation が無い場合だけ対象にする。
  - call の return effect slot に fresh `#a` で `Empty-subtract(γ, #a)` を登録する。
  - `#a` は現在の定義 lambda frame に記録し、その lambda の `ret_eff` と `ret` を
    `Pos::NonSubtract(_, #a)` で包む。
  - module/global callee には付けない。
  - anonymous lambda 内の application は外側の定義 lambda predicate へ上げない。
- `ExprLowerer::lower_binding_body_expr` を追加した。
  - 通常の `lower_expr` は anonymous lambda として扱う。
  - binding body 直下の lambda だけを、定義された関数の predicate を持つ lambda として扱う。
- 未注釈 parameter の function `arg_eff` は fresh var ではなく `Neg::Bot` にした。
  - 普通の関数は first cut でも、内部的に parameter effect が `never` だと分かる形へ寄せる。
  - `x: int` のような value-only annotation でも outer parameter effect は `never` のままにする。
  - `[handled] 'a` のような top-level effectful annotation だけ fresh effect slot を function `arg_eff` に出す。
- `poly::types::Neg::Bot` を追加した。
  - 単体表示は `never`。
  - function effect slot では省略する。
  - `a [never] -> [b] c` は `a -> [b] c`、`a -> [never] c` は `a -> c` と表示する。
- pure function subtyping の effect passthrough 規則を constraint 分解へ追加した。
  - `a -> [b] c <: d [e] -> [f] g` では、parameter effect を `e <: never` と比較しない。
  - 代わりに `e <: f` を登録し、純粋関数を通る引数側の効果を返り効果へ素通りさせる。
  - `e` と `f` はどちらも元の関数制約の upper 側にあるので、外側の right subtract weight を
    passthrough 制約の左右両方に載せる。

## Structural comparison completion

- `infer::constraints::ConstraintMachine` の同形 structural comparison を広げた。
  - `Con` の型引数は `Neu` から通常の lower/upper を作り、不変として両向き制約を登録する。
  - `Neu::Bounds(lower, var, upper)` は middle variable を消さず、positive 側を `lower | var`、
    negative 側を `var & upper` として作る。
  - `Neu::Con/Fun/Record/PolyVariant/Tuple` も、必要になった時点で通常の `Pos` / `Neg` bounds に戻す。
  - record / record spread / poly variant / row item の比較を通常の subtype 制約へ落とす。
- positive row は tail 付き表現へ戻さない。
  - concrete row item は matching upper item があれば型引数を比較し、なければ upper tail へ流す。
  - row variable item は negative row 全体を upper bound として受ける。
  - row 専用の保護 table や特殊 residual state は増やしていない。
- mismatched nominal `Con(A) <: Con(B)` は constraint machine で即 type error にしない。
  - user-defined cast / representation coerce が後段で説明できる境界かもしれない。
  - diagnostic 層を足す時も、`Con` mismatch は cast/evidence 解決を待ってから報告する。
- cast を別の後段補正だけにせず、型推論中に「cast 関数を適用したかのように」振る舞わせる案が出た。
  - `A <: B` が nominal mismatch で止まる境界に対して、`Cast A B` 相当の関数候補を探す。
  - 見つかった cast 関数 `A -> B` を、その境界で暗黙に適用した場合と同じ制約を追加する。
  - これにより user-defined cast も通常の関数型推論経路に寄せられ、cast だけ後段で型を書き換える形を避けられるかもしれない。
  - ただし source / target が同じ nominal head の `Path -> Path` 相当なら、型としては変化しないので cast 候補にしない。
  - 注意点: cast 候補探索で再帰・曖昧性・missing impl diagnostic をどう扱うかは別途設計する。

## Level / extrusion first cut

- 新しい `infer::constraints::ConstraintMachine` に `TypeLevel` と type variable level table を追加した。
  - 未登録の手書き `TypeVar` は root level として扱う。
  - `infer::Arena::fresh_type_var` は current level を constraint machine に登録する。
  - `Arena::enter_child_level` / `restore_level` を追加し、lambda body lowering の間だけ child level に入る。
- extrusion は古い `yulang-infer` と同じくコピーではなく「老化」にした。
  - shallow variable の lower bound に deeper variable を含む `Pos` が入る直前で、含まれる変数を target level まで下げる。
  - shallow variable の upper bound に deeper variable を含む `Neg` が入る直前でも同じ。
  - `Pos` / `Neg` / `Neu` の構造をたどり、`Neu::Bounds(lower, var, upper)` の center var と上下界も対象にする。
  - 老化した変数の既存 lower / upper bounds も同じ target level へ再帰的に老化する。
  - bounds cycle では visited type var で止める。
- まだ generalize / freeze とは接続していない。
  - 今回は level 不変条件を bound 登録機構へ入れる足場。
  - 次に SCC close / scheme 化を作る時、quantify 対象は def boundary より深い level の変数として拾う。

## Open SCC use constraints

- `RefId -> DefId` / method selection target の解決時点では、量化済み target 以外は型制約を直接作らない方針を確認した。
  - 解決結果はまず `SccMachine` へ `UseResolved { parent, target, use_value }` として渡す。
  - target が未量化かつ別 component の場合は、component edge として保持する。
  - target が量化済みの場合だけ、従来通り `InstantiateUse` event にする。
- 未量化 target が同じ SCC 内に入った use を `SccEvent::OpenUse` として開くようにした。
  - すでに同じ component にいる use は、その場で open する。
  - 新しい cycle merge で内部化した edge は、捨てずに `OpenUse` に変換する。
  - target root がまだ登録されていない場合は、component 内の pending open use として保持し、`RegisterDef` 時に開く。
- `AnalysisSession` は `OpenUse { target_root, use_value }` を受けて、同じ SCC 内の参照として片方向制約を追加する。
  - `target_root <: use_value`
  - この edge だけで、target の lower bound は use へ、use の upper bound は target へ伝播する。
  - 逆向きの `use_value <: target_root` まで入れると equality に近くなり、use-site の lower bound まで def root へ戻してしまう。
  - SCC machine は constraint machine を直接触らず、event 境界を保つ。
- 自己再帰は最初から open できるように、binding body lowering 前に def root slot を作って `RegisterDef` を queue へ積むようにした。
  - `RefId` の `ApplyRefResolution` より前に root 登録が queue に入るため、`my f = \x -> f` の self ref は pending root を待たずに `OpenUse` になる。
  - finish 時には def root と body value を同一定義として両向き制約で結ぶ。これは use-site ではなく binding 定義本体の同一性。

確認:

- `cargo test -q -p infer -- --nocapture`

## SCC quantification / instantiation first cut

- `SccEvent::QuantifyComponent` を `AnalysisSession` で受け、各 `Def::Let.scheme` へ compact generalize 済みの `poly::Scheme` を書き込むようにした。
  - `compact` の極性消去・共起分析・sandwich 結果を `GeneralizedCompactRoot` として保持し、finalize 時には ancestor の代入情報を子孫側へ反映する。
  - scheme を書いた後も compact 表現は `AnalysisSession` 側に残し、後続の子孫 finalize に使う。
- definition root の `TypeVar` は binding body lowering と同じ child level で発行するようにした。
  - root 自体を generalize 側で特別に消す処理は入れない。
  - root も通常の child-level variable として、極性消去や量化対象判定に乗せる。
- `SccEvent::InstantiateUse` を `poly::Scheme` の fresh 展開として実装した。
  - scheme predicate を `poly::TypeArena` から infer arena へ clone し、quantifier は use-site level の fresh var に置き換える。
  - subtract id も fresh にし、同じ写像で `Pos::NonSubtract` と subtract fact を復元する。
  - 展開後の predicate は通常の subtype 制約として use-site value slot へ戻す。
- scheme instantiation の copier は `infer::instantiate` に分け、`analysis` は SCC event routing と scheme 書き込みだけを見る形にした。

### Weight-insensitive compact simplification follow-up

- compact の極性消去と共起分析は subtract weight を見ない方針へ寄せた。
  - 共起分析の bucket key は `TypeVar` の共起 group だけで作り、weight が違う変数も同じ構造なら統合する。
  - 統合で同じ target var に複数 occurrence が潰れた場合は、`ConstraintWeight` を union して target occurrence へ載せる。
  - 反変側に出ない変数は極性消去で落ちるため、その変数だけに載っていた subtract weight も自然に消える。
- generalize の subtract 収集でも simplification substitution を見るようにした。
  - `T-subtract(α, #id)` があり、共起分析などで `α -> β` になった場合は、scheme 側では `T-subtract(β, #id)` として扱う。
  - instantiate 時は `SubtractId` から元 fact を引けるよう、`SubtractTable::fact_by_id` を追加した。
