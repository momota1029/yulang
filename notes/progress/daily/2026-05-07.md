# 2026-05-07

## Compiled Runtime Surface

- Started Phase 4 of the compiled-unit cache path with a conservative
  `CompiledRuntimeSurface`.
- `CompiledUnitArtifact` now bundles syntax, namespace, typed, and runtime
  surfaces under the same unit manifest.
- The runtime surface currently stores a unit-local `CoreProgram` exported from
  the namespace value symbols that belong to the unit.
- `import_compiled_unit_artifact` returns the runtime surface alongside restored
  syntax and typed-ref coverage. This is still a read-only artifact checkpoint;
  it does not yet merge dependency runtime surfaces into a downstream user core
  export.
- Wasm now exposes `std_compiled_unit_artifacts()` so playground build-time
  tooling can inspect the unit-oriented artifact shape instead of the older
  std-only core snapshot shape.
- The wasm artifact export is intentionally not on the normal playground run
  path. It is a persistence/bundling checkpoint for moving std work out of the
  first interactive compile.
- Added `CompiledRuntimeBundle::from_surfaces` as the first runtime/core merge
  checkpoint:
  - dependency core bindings are deduplicated by canonical path;
  - conflicting same-path bindings or runtime symbols are rejected;
  - expected-edge evidence ids are remapped across merged surfaces, including
    references inside binding bodies and principal plans.
- Added `CompiledRuntimeBundle::merge_with_user_program` so cached dependency
  runtime surfaces can be placed before a freshly exported user program.
- Added a wasm regression that removes a dependency binding from the fresh user
  core export, restores it from a compiled runtime surface, then lowers,
  monomorphizes, compiles, and evaluates the merged runtime program.
- Runtime surfaces are now seeded from all binding paths owned by the unit's
  module paths, not just namespace value exports. This keeps generated same-unit
  runtime helpers such as variant constructors while avoiding reexport-owned
  duplicates from other units.
- Added a std-facing wasm regression that bundles source-std runtime surfaces,
  removes those bundled bindings from the fresh program export, merges the
  bundle back in, and evaluates `1 + 2`.
- Added a `crates/yulang-wasm` build script that builds std compiled-unit
  artifacts at package build time and embeds the serialized JSON from `OUT_DIR`.
- Wasm now exposes `embedded_std_compiled_unit_artifacts()` and
  `embedded_std_compiled_unit_artifact_status()`.
- `warm_std_cache()` includes embedded artifact counts, runtime binding counts,
  and artifact byte size in its console-visible warmup output.
- The std runtime-surface merge regression now uses the embedded artifacts
  rather than rebuilding artifacts inside the test body.
- Embedded artifact loading now validates manifest format versions plus
  namespace/typed surface completeness. Status reporting returns `valid=false`
  and a fallback reason instead of panicking, so the normal source compile path
  can remain the conservative fallback.
- The playground logs embedded std artifact status immediately after wasm init,
  and warmup logs include the same cache-artifact status fields.
- Typed surfaces now serialize `CompactTypeScheme` payloads for exported unit
  value schemes instead of only carrying rendered debug strings.
- Typed import installs imported compact schemes into the restored `Infer`
  state and rebuilds frozen schemes from those compact payloads, so downstream
  refs can instantiate cached dependency schemes without re-lowering the cached
  source body.
- Added a multi-unit compiled artifact import path that merges namespace/typed
  surfaces, imports them into one `LowerState`, and carries a merged runtime
  bundle alongside the lowered user state.
- The playground run path now tries embedded std compiled-unit artifacts before
  the process-local source std cache. On a hit it skips std source lowering and
  merges embedded std runtime surfaces into the fresh user core program.
- Artifact execution keeps a conservative fallback: if import, diagnostics,
  runtime merge, VM lowering, monomorphize, compile, or eval fails on the
  embedded path, the run retries through the existing source std cache and
  records the fallback reason in timings.
- The fallback also triggers if embedded runtime execution leaves a root request
  unresolved. This keeps current console/effect/role gaps from changing user
  visible playground results while typed-surface import is still incomplete.
- Namespace surfaces now include same-unit companion modules, fixing imported
  paths such as `std::prelude::Add::add` without pulling unrelated root children
  into entry artifacts.
- Runtime bundle merge now prunes user-exported duplicate dependency runtime
  bindings/symbols before merging cached dependency surfaces.
