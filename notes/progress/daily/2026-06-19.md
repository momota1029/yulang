## control VM arm payload lookup

- 残っている `shared_frame_unwrap_clones` のうち `case` / `catch` bucket を見た。
  `examples/showcase.yu` の単発 debug timing では
  `case=1655`、`catch=3285`、`block=966`、`apply=831`。
- `CaseGuard` / `CatchValueGuard` / `CatchRequestGuard` だけを borrowed execution に載せる案を試したが、
  `shared_frame_unwrap_clones` が動かず、該当 clone は guard frame 由来ではなかったため戻した。
- `BindThen::CaseArm` / `CatchValue` / `CatchRequestPayload` が共有済み arm table の要素を
  さらに所有していたため、`arm` payload を外した。
  bind 後は `arms + index` から `guard` / `body` / `continuation` を読む。
  `eval_case_arm` / `handle_catch_*_arm` も `arms[index].clone()` せず、
  `bind_pat` に必要な pattern だけ clone する。
- repeat 5 の `bench/nondet_20_discard.yu` release / no-cache:
  `runtime_execute` 56.9ms / 56.3ms / 59.9ms / 56.3ms / 54.8ms。
- repeat 5 の `examples/showcase.yu` release / no-cache:
  `runtime_execute` 126.9ms / 122.5ms / 128.7ms / 122.7ms / 122.9ms。
  `shared_frame_unwrap_clones` は変わらないため、これは arm payload clone 削減として見る。
