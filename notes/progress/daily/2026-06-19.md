## language server diagnostic declaration ranges

- LS の diagnostic がすべて `Range::default()` に出ていたため、まず compiler 側から
  structured range を流す入口を追加した。
- `sources::SourceRange` を追加し、pass1 の module registration 時に
  `DefId -> declaration name range` を `ModuleTable` へ保存するようにした。
  通常 binding / operator / companion method / act operation / enum variant / cast /
  root synthetic owner など、DefId を持つ lowering diagnostic の owner range を取れる。
- `SourceDiagnostic` に `range: Option<SourceRange>` を追加し、
  `CheckDiagnostic.def` から宣言 range を引いて source diagnostic へ載せる。
- LSP 側では compiler の byte range を UTF-16 `Range` へ変換し、
  root source に注入される implicit prelude / `mod std;` の offset を引いて
  open document 座標に戻す。
- これは「use-site primary range」の完全対応ではない。
  `LoweringError` 自体にはまだ expression/token span がないため、
  unresolved name などは当面 owner declaration range に出る。
  本命は lowering error を spanned 化して、diagnostic の primary range をそのまま流すこと。

確認:

- `cargo test -q -p yulang server -- --test-threads=1`
- `cargo test -q -p yulang source::tests::case_02::analyze_entry_source_uses_in_memory_root_source -- --test-threads=1`
- `cargo check -q -p yulang`
- `cargo test -q -p infer lowering -- --test-threads=1`

## language server unresolved-name primary ranges

- 前段の declaration range fallback に続き、`LoweringError::UnresolvedName` に
  `source_range: Option<SourceRange>` を持たせた。
- expression lowering で source token から来る identifier / sigil identifier / path prefix /
  operator use の range を `lower_*_at` 経由で渡すようにした。
  synthetic std lookup や synthetic act member lookup など、source token がない未解決名は
  `source_range: None` のままにする。
- `SourceDiagnostic` の range は、lowering error 自身の source range を優先し、
  ない場合だけ `DefId -> declaration name range` に fallback する。
- これで `my result = missing` の LSP diagnostic は `result` 宣言ではなく
  `missing` token に出る。型 mismatch など、まだ source range を持たない診断は
  declaration range fallback のまま。

  replay の意味は変えず、小さい lower/upper replay で heap allocation を避ける。
- `generalize_root_with_prepasses` の開始時に
  `applied_method_role_resolutions` と demand set を全 def で clone していたが、
  role 入力が最初から存在する def だけで作るようにした。
  role 入力なしの def は role resolution / residual role 挿入に入らないため、固定費だけを削る。
- `quantify_component` の finalize phase で使う `def_parent_map` は、
  component ごとに poly defs 全体を再走査せず、`DefArena::len()` が変わるまで cache するようにした。
  scheme を埋めるだけでは def 数は変わらないため、祖先探索の入力 map を共有できる。
- 前段で入れた compact shape counter は tree walk が重いため、
  `YULANG_GENERALIZE_SHAPE_TIMING=1` の時だけ有効にした。
  通常 timing では shape counter は 0 になり、playground の hot path へ乗らない。
- `examples/showcase.yu` の release / infer-only repeat 5 では、
  `infer` は概ね `230〜238ms`、`constraint.drain` は `78〜82ms`、

## language server type-only hover

- LS hover は doc comment / Yumark rendering を混ぜず、まず inferred type だけを返す形にした。
- compiler 側では declaration range を `DefId -> SourceSpan { file, range }` に広げ、
  root/std/local module の同じ byte offset が混ざらないようにした。
  `RefUse` にも source token span を載せ、hover 位置から reference target の scheme を引ける。
- source route に `hover_entry_source_with_std_options` / `hover_entry_source` と
  `SourceHover { range, contents }` を追加した。
  range は既存 diagnostics と同じ loaded root 座標なので、LSP 側は implicit prelude/std module
  prefix を引いて open document 座標へ戻す。
- LSP は `hoverProvider` を有効化し、hover response は Markdown の `yulang` code block で
  `x: int` のような型だけを返す。
  UTF-16 position から byte offset への逆変換も追加し、surrogate pair の途中は無効位置として扱う。
- 今回は `Def::Let` の generalized scheme を持つ対象だけ表示する。
  local lambda arg など `Def::Arg` の use-site 型表示は、後で `TypeVar` 表示 API を compiler 側へ
  きれいに出してから広げる。
