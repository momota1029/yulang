# 2026-05-24

## Runtime finalize instance cache surface

既存 compiled-unit 永続 cache へ runtime-finalize instance を載せる前段として、
`yulang-runtime-finalize` 内に cache surface を追加した。

- `FinalizeInstanceKey`: `binding + type substitutions`
- `CachedFinalizeInstance`: monomorphic `scheme` / `body` / `callee_type` / `result_type`
- `FinalizeInstanceCacheSurface`: serde 可能な永続化候補 surface
- `FinalizeInstanceCache`: in-memory cache + policy + profile

`finalize_module_with_cache` は、append 時に cache hit した instance の materialized body を再利用する。
miss 時は従来どおり graph solve / materialize を行い、policy に通る instance だけ cache に入れる。
肥大化対策として、default policy は最大 4096 entries、1 body あたり最大 2048 expr nodes を上限にする。
これは persistent artifact へ保存する前の safety valve で、cache は advisory のため skip しても意味は変わらない。

さらに `FinalizeInstanceArtifactCache` を追加し、既存 compiled-unit cache と同じ
`CompiledUnitManifest` set を key にして、runtime-finalize instance surface を sidecar file として読めるようにした。
path は既存の `compiled-units/v*/parser-v*/` 配下に `runtime-finalize-v*/instances-*.bin` を作る。
infer の compiled-unit artifact 本体に runtime-ir body を混ぜると層が逆流するため、artifact 本体は触らず、
同じ manifest identity を使う隣接 cache として扱う。
呼び出し側は `read_cache_for_manifests` / `write_cache_for_manifests` で
`FinalizeInstanceCache` を直接 load/store できる。

追加 regression:

- `my id x = x; id 1` を同じ `FinalizeInstanceCache` で 2 回 finalize すると 2 回目は cache hit する。
- `FinalizeInstanceCacheSurface` へ出したあと、復元した cache からも同じ instance が hit する。
- 実際に finalize して作った `FinalizeInstanceCache` を `FinalizeInstanceArtifactCache` へ書き、
  manifest key から読み戻した cache で solver が同じ instance を hit する。
- fake std manifest set で `FinalizeInstanceArtifactCache` の write/read が roundtrip する。
- compiled artifact/parser format が混ざった manifest set は reject する。

未解決:

- `"hello".println` を std ありで `runtime-finalize` まで通す通常スタック test は、現状の
  finalize instance cache だけでは成立しない。compiled std artifact を使って infer/core
  merge までは通常スタックで進むが、その後の `yulang_runtime::lower_core_program` が
  stack overflow する。これは finalize cache 以前の core -> runtime IR lowering cache が
  まだないため。

確認:

```bash
RUSTC_WRAPPER= cargo test -q -p yulang-runtime-finalize finalize_instance_cache_surface_reuses_materialized_binding -- --nocapture
RUSTC_WRAPPER= cargo test -q -p yulang-runtime-finalize finalize_instance_artifact_cache_rehydrates_solver_cache -- --nocapture
RUSTC_WRAPPER= cargo test -q -p yulang-runtime-finalize artifact_cache_uses_compiled_unit_manifest_key -- --nocapture
RUSTC_WRAPPER= cargo check -q -p yulang-runtime-finalize
cargo fmt --all --check
```

はいずれも通る。
