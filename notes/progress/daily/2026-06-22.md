## Directed stack weight proof

- Read `research/effect-mini-language/directed_stack_weight_principal_letrec_colored_soundness_ja.tex`.
- The colored soundness draft makes the implementation target sharper:
  - left and right weights are distinct structures;
  - row split consumes only `J = K ∩ Common(L)`;
  - `protect` is `PWeight(take(Empty), rho)`;
  - filter is a static check and has no runtime marker;
  - soundness is for normalized, well-bracketed colored marker elaboration.
- Wrote implementation notes in `notes/design/2026-06-22-directed-stack-weight-implementation.md`.
- Immediate implementation risk:
  `crates/infer/src/constraints/row_effect.rs` combines `weights.left.compose(&weights.right)` before row split,
  which conflicts with the directed pair interpretation.
- Started implementation slice 1 by adding `crates/infer/src/constraints/directed_weight.rs`.
  This module is not wired into the solver yet; it fixes the new same-id monoid and W-Mix behavior with unit tests.
- Ported `crates/infer/src/constraints/row_effect.rs` to the directed row split contract:
  - no left/right composition before row-head subtraction;
  - row head uses `Common(L)` from left active stack entries only;
  - filters are checked and erased before residual propagation;
  - right-side pop is carried to the residual tail and mixed only after split;
  - row residual tests no longer expect legacy `floor` residuals.
- Moved W-Mix normalization into `ConstraintWeights` replay composition:
  - bound replay now composes weights, applies directed W-Mix, then applies the implementation-side pop cap;
  - var-var replay applies alias pop caps after W-Mix;
  - `row_effect.rs` now reuses the shared normalization instead of a local helper.
- Updated `Neg::Stack` absorption so filter is checked and then erased instead of being stored as a right weight.
- Split `ConstraintWeights.right` to a dedicated `RightStackWeight`:
  - right weights now structurally store only pure pops;
  - compact collection materializes right pop weights explicitly with `to_stack_weight()`;
  - alias-neutral and row-tail-boundary checks no longer treat right weights as carrying family budgets;
  - tests no longer construct impossible right push/floor/filter weights.
- Moved left-side filter handling out of stored bound weights:
  - lower-bound insertion checks the weighted lower type and erases the left filter before storing;
  - upper and var-var bound insertion register the filter on the source/lower variable, then store filter-free weights;
  - replay and compact paths no longer see left filters that have already crossed a variable boundary.
