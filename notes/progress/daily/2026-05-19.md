# 2026-05-19

- Added an MMTk allocation profile for functional-language GC tuning probes.
  `MmtkHeap` now records total object count, trace slots, raw payload bytes,
  total object bytes, max trace slots, max raw payload bytes, object-kind
  buckets, and compact storage-shape buckets as allocations happen.
- Generated MMTk executable harnesses can dump the profile with
  `YULANG_MMTK_CPS_HEAP_STATS=1`. The lines are prefixed with
  `[JIT-MMTK-STATS]` and are no-ops unless the env var is set.
- First 20x20 readings:
  - `.list.len`: heap objects=1,026, trace_slots=4,998; list dominates with
    899 objects and 4,872 trace slots.
  - `.list.say`: heap objects=12,459, trace_slots=58,165; string has 5,984
    objects and list has 4,748 objects, with list carrying 46,903 trace slots.
- Started the CPS optimization iteration loop with an implementation note at
  `notes/design/cps-optimization-pass-plan.md`.
- Added unused continuation environment slot trimming. The pass accounts for
  continuation entries referenced by closure/thunk creation, handler envs, and
  resume targets before removing environment slots, then reindexes remaining
  slots. The 20x20 traces show `removed_unused_environment_slots=1` for
  `.list.len` and `=2` for `.list.say`; generated-executable hyperfine does not
  show a clear wall-clock win from this small pass alone.
- Added `\case:` / `\catch:` shorthand parsing and lowering. The syntax is
  restricted to no-space `\case` / `\catch`, then lowers to a one-argument
  lambda whose receiver becomes the existing `case` scrutinee or `catch`
  computation. Parser and VM regressions cover both forms.
- Added labeled recursive `case` / `catch` shorthand:
  `case 'label x:`, `catch 'label action:`, `\case 'label:`, and
  `\catch 'label:`. The labeled expression forms lower to a recursive
  one-argument function applied to the target expression; the lambda forms
  return that recursive function. Catch labels are tested with shallow-handler
  resume behavior by explicitly calling the label again after `k`.
- Added the `...` YadaYada operator as a nullfix expression. It lowers to a
  `never`-typed primitive, type-checks in unreachable positions, and raises a
  VM error when evaluation reaches it.
- Started the parser-combinator stdlib surface for `rule` / `~"..."` desugar.
  `std::parse` now defines the thin `[parse]` effect kernel (`item`,
  `position`, `fail`, `commit`) plus first-order combinators for `choice`,
  `read`, `satisfy`, optional parsing, greedy repetition, and lazy repetition.
- Added the first `std::parse::run` / `run_full` handler slice. It handles
  `item`, `position`, `fail`, and `commit`, returning
  `result (value, position) parse_error`. `choice` is an ordinary combinator
  built with a local shallow handler that tracks whether the current parser
  branch committed or consumed input. It retries the right branch only when the
  left branch fails without consuming input and without commit; committed
  failures propagate to outer choices.
- Reported and fixed a recursive local handler specialization bug. A program
  with two local variable handlers and a recursive function could specialize
  the same `run : init -> result -> result` handler with a stale contextual
  output (`bool`) while the input continuation result forced `result = int`.
  Principal-unify now refines contextual specializations from input shapes
  before accepting output hints, ignores conflicting output hints when inputs
  determine the result, carries those shapes through active recursive rewrites,
  and refreshes rebuilt apply arguments before using them as specialization
  evidence. Added a VM regression for the two-handler recursive case.
- Added a first-class `char` surface while keeping the VM representation as a
  one-grapheme `StringTree`. `str[index]` now has type `std::char::char`;
  `std::char` exposes equality, `to_string`, whitespace, punctuation, and broad
  word-token predicates.
- Switched that `char` surface from Unicode scalar values to extended grapheme
  clusters. `StringTree` now counts/indexes/slices by grapheme clusters while
  retaining byte length metadata. MMTk compact string helpers use the same
  grapheme boundary rules.
- Extended `std::parse` with `peek`, `token`, and `word`. `item` now returns
  `char`; `read` remains as a `token` alias. `word` uses `peek` so it can stop
  before punctuation or whitespace without consuming the delimiter. VM
  regressions cover char `item`, non-consuming `peek`, `token`, and broad
  Unicode word parsing.
- Added the first parser-sugar lowering slice. Plain `~"abc"` and a single-token
  `rule { "abc" }` lower to parser values equivalent to
  `\() -> std::parse::token "abc"`. Unsupported rule-literal capture /
  interpolation and richer rule bodies intentionally do not become successful
  parsers yet; they lower to the existing `...` runtime-error primitive until
  the full combinator desugar is implemented.
- Extended the parser-sugar slice with capture-only `~":name"`. It lowers to a
  parser value that runs `std::parse::word()`, binds the result, and then returns
  `{ name: value }`, so effectful parsing stays outside record-field lowering.
- Generalized rule-literal capture lowering to mixed token/capture sequences
  like `~":category/:article/index.html"`. The desugar runs each token or word
  parser in source order, then returns a record containing the captured fields.
- Split `std::parse::run` from positional prefix parsing. `run_with_pos` keeps
  the old `(value, pos)` result, while `run` now requires full consumption and
  returns only the parser value. `run_full` remains as a compatibility alias.
- Extended `rule { ... }` lowering so string literals are direct parser items
  and `name = parser` captures the parser's returned value into the final
  record.
- Added the first `rule { ... }` PEG operators. Alternation lowers to
  `std::parse::choice`, and `+` / `*` plus lazy and optional variants lower to
  the stdlib repetition combinators. Literal `token` now checks with `peek`
  before consuming, so failed first-character alternatives can backtrack.
