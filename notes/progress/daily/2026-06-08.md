## Elaborated IR boundary

- `yulang-elaborated-ir` crate を追加した。
  - `yulang-elaborate` が `InferExport` から組む、単相化済み・型付き output IR の data crate。
  - algorithm は持たせず、runtime-lower が読む型 surface だけを置く。
- concrete type wrapper を追加した。
  - `ConcreteType::try_from_type` は `Unknown` と free type variable を拒否する。
  - `Any` / `Never` は Top / Bottom として concrete type に残せる。
  - serde deserialize も検証経路を通す。
- `EffectiveThunkType` と `ForceThunk` node を追加した。
  - effect を持つ thunk は `MonoType::EffectiveThunk` として表す。
  - thunk open は旧 `BindHere` ではなく、`source effective-thunk -> target computation`
    の boundary として `ForceThunk` に残す。
  - `Cast` と同じく、producer-consumer の型差が確定した場所で elaborated IR に明示する。
- `yulang-elaborate` から `yulang_elaborated_ir` を `elaborated_ir` として re-export した。

## Elaboration constraint bounds graph

- `yulang-elaborate` の `ConstraintSolver` を、`DraftComputationId -> MonoComputation` の
  代入表から `TypeNode -> { lower, upper }` の制約グラフへ変更した。
  - computation value / effect と、scheme instantiate で fresh 化した型変数を同じ graph node として扱う。
  - `Type::Var(a) <: Type::Var(b)` は subtype edge、`T <: a` は lower bound、`a <: T` は upper bound。
  - lower 追加時は既存 upper へ、upper 追加時は既存 lower へ `constrain_types` を流す。
  - edge は lower を上流から下流へ、upper を下流から上流へ伝播する。
- generic direct ref apply は、量化変数を use-site fresh node に置き換えてから、
  actual arg / expected result の上下界で解くようにした。
  - `id : α -> α` / `id 1 : int` は `int <: α` と `α <: int` から `α = int` を読む。
  - まだ concrete 候補が一意に得られない場合は `GenericDirectRefScheme` で止め、`Any` / `Unknown`
    へ逃がさない。
- first cut の構造分解は function / tuple / 同一 named constructor の type args まで。
  record / variant / row / role obligation の本格制約化と、cast boundary 生成は次の作業。

## Structural expression elaboration

- `yulang-elaborate` の solver で record / variant / select / pack を扱うようにした。

## Function value runtime shape

- `spec/2026-06-07-principal-monomorphization.md` に、concrete function type を
  runtime shape へ読む規則を明記した。

  - `Never` は empty effect row なので、`thunk[Never, a]` は作らず bare value shape に残す。

  - `EffectiveThunk` と同じく erased `Type` graph に戻せない shape として、
    `yulang-elaborate` の explicit computation assignment table へ載せられるようにした。
- まだ solver output の全 computation をこの shape へ自動変換する段階には進めていない。

## Runtime-shaped apply boundaries

- solver 内部は erased `Type::Fun` のまま解き、elaborated IR 出力時に
  `MonoComputation::into_runtime_shape` で concrete function 型を `FunctionBoundary` へ変換するようにした。
  - `ProgramBuilder` の specialization key は raw solver signature のまま保持する。
  - materialized `MonoInstance.signature` は、実際に出力される runtime-shaped computation を使う。
- apply elaboration で callee の `FunctionBoundary` を読むようにした。
  - parameter shape が `EffectiveThunk` の場合だけ、argument computation を `MakeThunk` で閉じる。
  - return shape が `EffectiveThunk` の場合だけ、inner `Apply` の pure 結果を `ForceThunk` で開く。
  - `Never` effect boundary は thunk 化せず、pure `int -> int` のような apply は bare `Apply` のまま残す。
- lambda parameter type も raw function type から同じ runtime shape 変換を通して決めるようにした。
  - pure parameter は bare value type。
  - effectful parameter は `EffectiveThunk`。
- `spec/2026-06-07-principal-monomorphization.md` の function runtime shape も、
  `thunk[Never, a]` を作らない規則へ明示した。

## Nested apply solving

- `yulang-elaborate` の apply solver で、callee が nested `Apply` の場合も、
  outer result computation と known argument computation から callee function computation を復元するようにした。
  - `f 1 2` の外側 apply が `UnsupportedApplyCallee(Apply)` で止まらない。
  - inline lambda apply も literal 限定ではなく、`known_computation` が読める argument を使う。

## Unbounded monomorphic defaults

- `spec/2026-06-07-principal-monomorphization.md` に、境界から concrete candidate が出ない fresh
  単相変数の default を明記した。
  - value type は `unit`。
  - effect row は `Never`。
  - `Any` / `Unknown` は引き続き fallback として使わない。
- `yulang-elaborate` の constraint solver に `UnboundedTypeDefault` を追加した。
  - scheme freshening 時に、量化 type variable は value default、量化 effect variable は effect
    default を記録する。
  - bounds が完全にない fresh 変数も substitution に入るように、default を持つ変数を
    `build_substitution` の走査対象へ追加した。
  - computation slot 自体に bounds entry はあるが concrete candidate がない場合も、value は `unit`、
    effect は `Never` を選ぶ。
- root effect 復元用の軽量 `TypeInstantiation` も同じ default を持つようにした。
- root `TypeBounds` から最初の concrete computation を作る入口でも同じ default を使うようにした。
  - まず既存の concrete candidate を探し、候補がない場合だけ structural candidate に `unit` / `Never`
    を補う。
  - `Union(Var, Fun(...))` のような bounds では、裸 `Var` を default 候補として混ぜず、
    構造を持つ候補を優先する。
- direct ref が apply callee ではなく値として現れる場合も、期待 computation から generic scheme を
  単相化して instance demand を作るようにした。
  - actual infer export の `my id x = x; id` は、unbounded input/output を `unit` に default した
    `unit -> unit` instance として materialize される。
- local `Def` を callee にする apply も、outer result と argument computation から callee function
  computation を復元するようにした。
- generic direct ref apply の instantiation は、argument computation が先に読めない場合でも、
  return/result 側の制約から parameter computation を concrete 化できるなら進めるようにした。
  - actual infer export の `my call f = f 1; my id x = x; call id` は、`id` を `int -> int`
    instance として materialize する。

## 推論パイプライン作り直し（poly 路線・新戦線）

上の elaborated IR 戦線（Codex）とは別系統。Codex 製の `yulang-elaborate` / `yulang-infer` が
「自分で全容を把握できない」ため、推論アルゴリズムごと自作で置き換える方針。`yulang-` prefix なしの
新 crate 群（`parser → poly → mono → vm` の一直線依存）。旧 `yulang-*` は当面併存、最終 archive。
詳細計画は memory の `poly-rewrite-plan` / `poly-cache-design`。

### poly crate（中心 IR の器）

- `crates/poly`。型 `types.rs` は既存資産をそのまま流用（Pos/Neg/Neu 極性分離＋hash-cons、
  `Scheme` に subtracts、`Neu::Bounds` = sandwich）。
- `expr.rs` に Arena を追加した。
  `Arena { roots: Vec<DefId>, defs: DefArena, refs, selects, expr, pat, type_ids, typ }`。
- Def は `DefArena`(FxHashMap<DefId,Def> + next, fresh/set/get) に置く。**登録し直し**（fresh で採番 →
  子・エイリアスを読みながら新 Def を組んで set）で in-place mut borrow を回避する。
- `Def::Let { vis, scheme: Option<Scheme>, body: Option<ExprId>, children }` に scheme/body を内包
  （並走テーブルは作らず「Def を見れば揃う」）。ほかに `Def::Mod { vis, children }` / `Def::Arg`。
- expression / pattern の入れ子は Arena ID で結ぶ形へ揃えた。
  - `Expr::Lambda(PatId, ExprId)`
  - `Pat::Con(RefId, Vec<PatId>)`
  - regression test: `poly::expr::tests::nested_expr_and_pattern_edges_use_arena_ids`
- field / method selection は `SelectId` を介す形にした。
  - `Expr::Select(ExprId, SelectId)`
  - `Select { name, resolution: Option<SelectResolution> }`
  - `SelectResolution::{RecordField, Method, TypeclassMethod}`
  - regression test: `poly::expr::tests::selection_sites_have_arena_ids_and_late_resolution`
- 原則: **DefId 紐づきは Def の中 / RefId 紐づきは refs テーブル**。id は Arena ごとに 0 から
  （static mut 廃止＝キャッシュ整合の布石）。
- `Vis` = Pub/Our/My（既定 Our＝band 内可視）。

### parser header_only 先読み

- 目的: ファイル別 op テーブルを組むため、body 前に use/op 宣言だけ読む軽量ヘッダパース。
- 停止点 = `op_def.rs` の `update_op_table()` 直後（symbol/fixity/bp 登録済み・body 事故ゼロ）。
- `context.rs` に `Env.header_only`、`op_def.rs` に `skip_op_def_body`、`stmt/mod.rs` に `HeaderStep` /
  `parse_header_statement` / `parse_header_visibility`、`lib.rs` に `parse_header_to_green`。
- 規約「export したいなら先頭に書く」とパーサ挙動が一致（最初の定義文で停止＝先頭 export op だけ拾い、
  中程ローカル op は無視）。

### sources crate（旧 sources を捨てて新規）

- 旧 sources はファイル別 op テーブルを使わず（`standard_op_table` 固定）、相互 use で op を
  取りこぼす不良品だった。
- 新フロー（**SCC 不要**）: 先読み(`read_header`) → 各ファイルの実効 export op を**不動点で閉包**
  （pub use 再エクスポート連鎖、op 集合は単調増加で収束） → ファイル別初期テーブル(standard + 輸入
  export op) → `parse_module_to_green_with_ops` でフルパース。全段階が順序無関係。
- `LoadedFile { module_path, source, header, op_table, cst }` を返す。
- テスト緑（3）。特に **cyclic_use_resolves_ops_both_ways**（循環 use で op が両方向に行き渡る＝
  旧 sources の不良＝作り直しの動機を修正できたことの実証）。

### infer pass1（CST → モジュールマップ）

- `crates/infer`。`lower_module_map(root) -> Lower { arena, modules }`。
- `register_block`: Binding を DefId 採番（`Def::Let{scheme:None, body:None}`）、ModDecl を採番＋子へ
  再帰（IndentBlock/BraceGroup）、UseDecl を `ModuleNode.aliases` に `(UseImport, Vis)` で溜める。
- `ModuleTable`（infer 作業用、poly には残さない）: Name→DefId / Name→ModuleId / aliases / parent。
- body・型制約には踏み込まない（pass2 の仕事）。
- テスト緑（3）: registers_top_level_bindings / registers_nested_module / collects_use_aliases。

### infer typing / weighted constraint footing

- `crates/infer/src/typing.rs` を追加した。
  - `Computation { expr, value, effect, evaluation }`
  - `Computation` は lowering の戻り値として引き回す一時値。`ExprId` と、その時点の
    value / effect / evaluation を一緒に持つ。
  - `Evaluation::{Value, Computation}`
  - `Typing` は永続的な型 table なので `DefId` の型だけを保持する。
  - `RefId` の parent / use-site type は `RefUseTable` に移した。
  - `ExprId` / `PatId` / `SelectId` には永続的な型を持たせない。式の型は lower 中の
    `Computation` に載せ、selection は `SelectId` の解決結果と、その場の constraint で扱う。
  - effect row と、値制限用の expansive 判定は分けて持つ。
- `crates/infer/src/constraints.rs` を追加した。
  - `SubtypeConstraint { lower, upper, weights }`
  - `ConstraintWeights { left, right }`
  - `ConstraintWeight` は `SubtractId` の sorted set。
  - `swapped()` / `union()` を最初から用意した。
  - passive な `ConstraintSet` ではなく、`ConstraintMachine` にした。
  - public `subtype` / `weighted_subtype` / `subtract_fact` は queue に work を積んで、その場で `drain()` する。
  - 伝播中に生まれた subtype work も同じ queue に積む。後で lowering と別 thread に分ける余地を残す。
  - `TypeArena` は `ConstraintMachine` が所有する。`poly::Arena` は最終 IR の器として触らない。
  - `TypeBounds` は `TypeVar` ごとに weighted lower / upper bounds を保持し、新しい bound は反対側 bounds と
    weight union で再伝播する。
  - `SubtractTable` は `S-subtract(α,#a)` facts を subtype bounds とは分けて保持する。
  - 新しい lower / upper bound と subtract fact が追加された時に `ConstraintEvent` を outbox へ出す。
  - `non_subtracts: Vec<(TypeVar, SubtractId)>` は削除した。subtract 抑制は独立 table ではなく
    weighted edge の左右 `SubtractId` 集合で持ち回る。
  - `ConstraintSite` は削除した。出自情報は制約の意味ではないため、型の出自として必要になった時点で
    constraint core の外側に設計する。単なる span table はまだ入れない。
- `crates/infer/src/arena.rs` を追加した。
  - `infer::Arena` は lowering / infer 中の作業 arena。
  - fresh `TypeVar` / `SubtractId` と `ConstraintMachine` を所有する。
  - `poly::Arena` とは分け、poly 側の最終表現を constraint 伝播の都合で揺らさない。
- `crates/infer/src/uses.rs` を追加した。
  - `RefUseTable`: `RefId -> { parent: DefId, value: TypeVar }`
  - `SelectionUseTable`: `SelectId -> { parent, receiver_value, method_value }`
  - selection watcher は `pending_by_receiver` と `pending_by_ref_payload` の2枠。
    `ref '[e] a` の payload `a` の下界が決まった時にも該当 selection を起こせる。
- `crates/infer/src/scc.rs` を追加した。
  - `SccMachine` は incremental component graph を持つ。
  - 普通の新規 edge は component 間 edge として張り、`ComponentEdgeAdded { from, to }` を出す。
  - 既存 component 間 edge が cycle を作る場合だけ関係する component を merge し、
    `MergeComponents { components, merged }` を出す。
  - method selection などで SCC と SCC がくっついた時に、「この components を互いに展開し合う」
    という安定したイベントを出すため、全体 Tarjan の作り直しにはしない。
  - target が量化済みなら edge を足さず、`SccEvent::InstantiateUse { target, use_value }` を出す。
  - closed な component が method / SCC 依存を持たなくなったら、SCC machine 自身が
    `QuantifyComponent { component, roots }` を出す。
  - 量化した component は open graph から削除し、そこへ入っていた use edge を
    `InstantiateUse { target, use_value }` に変換する。
- `crates/infer/src/analysis.rs` を追加した。
  - `AnalysisSession` は constraint events を selection / SCC work へ routing する coordinator。
  - `LowerBoundAdded(var)` で `SelectionUseTable` の2種類の watcher を見て `ProbeSelect` を enqueue する。
  - `ApplyRefResolution` / `ApplySelectionResolution(Method|TypeclassMethod)` は use-site `parent` と
    `value` / `method_value` を `SccMachine` へ渡す。
- ここでは solver / prune / handler subtract 解釈には入らない。
  pass2 lower が、あとで solver へ渡せる型 slot と weighted constraint seed を作るための足場。

- `cargo test -p poly -p sources -p infer`

### キャッシュ方針（決定）

- 「Arena とキャッシュは相性が悪い」の正体は Arena の ID（DefId/ExprId…）が位置依存で安定しないこと。
  → **ID をキャッシュ境界に晒さない**。Arena は揮発の作業メモリと割り切り、丸ごとシリアライズしない。
  境界は generalize 済み Scheme（公開シンボルの主型）で繋ぐ。
- 単位 = realm 内 file-SCC。変化検出は **CST まるごとの雑ハッシュ**で弾く（early-cutoff も Scheme の
  内容アドレス正規化も作らない＝「SCC ごとに 入力ハッシュ→推論結果 の Map 一個」で済む）。
- **健全性の一点だけ必須**: キャッシュキーに依存先 SCC のハッシュを畳み込む（SCC は逆トポ順処理なので
  依存先の鍵は確定済み＝畳み込むだけ）。これを怠ると依存先シグネチャ変更時に stale ヒットする。
- 2レイヤ: poly キャッシュ（SCC インターフェース=型検査スキップ）/ mono・vm キャッシュ（具体型
  インスタンスの単相化コード、別層）。詳細は memory `poly-cache-design`。

### TypeId local counter

- `types.rs` の `fresh_type_var` / `fresh_subtract_id` グローバル関数を廃止した。
- `TypeIds` を追加し、`Arena` が `type_ids` として保持する形にした。
  - `Arena::fresh_type_var()` / `Arena::fresh_subtract_id()` から発行する。
  - ID は Arena ごとに 0 から始まり、プロセス全体の `static mut` に依存しない。
  - DefId/ExprId を Arena ローカルにした方針と揃え、将来の SCC キャッシュ境界へ ID を漏らさない。
- regression test:
  - `poly::expr::tests::type_ids_are_arena_local`

### pass2 段取り
- pass2 の縦串（薄く通す）:
  1. 式 lower + 名前解決 + typing / constraint seed
     - CST→`Expr`
     - `Var` に RefId
     - スコープを上→下で辿って DefId 解決
     - lower 結果として `Computation { expr, value, effect, evaluation }` を引き回す
     - 永続 table には `DefId` / `RefId` の型だけを作る
     - `SelectId` は selection site の解決先を持たせ、型はその場の computation / constraint で扱う
     - subtype constraint は weighted edge として記録する
     - まだ solver は走らせない
  2. constrain（biunification 本体。型変数に上下界を貯めて subtype を伝播）
  3. infer_expr（App/Lambda/Lit に型付け）
  4. generalize（let 多相・SCC・世代化）
- エイリアス解決は pass1.5 を作らず、pass2 で dot-selection の遅延機構に相乗り（名前の遅延／型の遅延、
  同 pass 別トリガー、循環は1周して何も進まなければ検出）。

### SCC machine quantification

- `crates/infer/src/scc.rs` を、open component graph が量化まで所有する形に更新した。
  - component は `members` / def root type vars / finished defs / method dependency count を持つ。
  - `RegisterDef` と `DefFinished` で、量化に必要な def 側の情報を埋める。
  - `MethodDependencyAdded` / `MethodDependencyResolved` で、pending method selection が残っている
    component の量化を止める。
- ready 条件:
  - 全 member def が root type var を登録済み。
  - 全 member def が lowering 完了済み。
  - method dependency count が 0。
  - open SCC への outgoing edge が 0。
- ready になった component は `QuantifyComponent { component, roots }` を出して open graph から削除する。
  削除時に incoming use edge は `InstantiateUse { target, use_value }` として吐き、predecessor component の
  outgoing edge が消えた場合は量化判定を cascade する。
- method selection などで新しい edge が cycle を閉じた場合は、全体 Tarjan ではなく
  `reachable_from(to) ∩ reaching_to(from)` の component だけを merge する。

確認:

- `cargo fmt -p infer -p poly --check`
- `cargo test -p poly -p sources -p infer`（sources 3 passed / infer 27 passed、poly 3 passed）

### Pattern name lowering footing

- `infer::patterns` を追加した。
  - `PatternLowerer` は pattern 名と payload 数だけを受け取り、constructor 参照か局所束縛かを決める。
  - `ConstructorTable` は `Name + arity` で constructor 候補を引く hook。enum / error / import 解決の実体は
    後段で登録する。
  - `PatternBindings` は pattern が導入した local `Def::Arg` を保持する。body resolver が local scope へ
    積むための table で、SCC edge は作らない。
- 判定規則:
  - constructor table に `Name + arity` があれば `Pat::Con(ref_id, payloads)` を作り、
    `poly::Arena` の `RefId -> DefId` を解決済みにする。
  - nullary constructor がある bare name は constructor を優先する。
  - constructor 名が存在しない bare name だけ、`Def::Arg` と `Pat::Var(def)` を作る。
  - payload を持つ名前が constructor として解けない場合や、既知 constructor 名の arity が違う場合は、
    暗黙の変数束縛へ落とさず error にする。
- まだ CST pattern lowering / diagnostics / typing constraints へは接続していない。

確認:

- `cargo fmt -p infer -p poly --check`
- `cargo test -p infer patterns -- --nocapture`（5 passed）
- `cargo test -p poly -p sources -p infer`（sources 3 passed / infer 32 passed、poly 3 passed）

### Expression lowering first cut

- `infer::lowering` を追加した。
  - `ExprLowerer` は 1つの owner def の body 用に作る expression lowerer。
  - 戻り値は `typing::Computation` で、`ExprId` / value slot / effect slot / evaluation を同時に返す。
  - 式ごとの型 table は作らない。
- 対応した CST:
  - integer / float literal。
  - 単一 `Ident` の module lexical lookup。
  - `ApplyML` / `ApplyC` の左結合 application。
  - `Paren` の unit / single expr / tuple。
- 名前参照:
  - `RefId` を発行し、`RefUseTable` に parent def と use-site value slot を記録する。
  - 解決結果は `AnalysisWork::ApplyRefResolution` として queue へ積む。
  - `AnalysisSession::drain_work` 側が `poly::Arena` への `RefId -> DefId` 書き戻しと SCC edge 追加を行う。
- application constraint:
  - callee value に `Neg::Fun { arg, arg_eff, ret_eff, ret }` upper bound を張る。
  - callee / arg / call effect は result effect へ流す。
  - 旧 `yulang-infer` の effect assumption / continuation / direct computation argument slot はまだ未移植。
- 未対応:
  - path selection / field selection / method selection。
  - operator / `with` / block / if / case / catch。
  - local pattern bindings と lambda lowering。
  - diagnostics span。

確認:

- `cargo fmt -p infer -p poly --check`
- `cargo test -p infer lowering -- --nocapture`（3 passed）
- `cargo test -p poly -p sources -p infer`（sources 3 passed / infer 35 passed、poly 3 passed）

### poly / infer design doccomments

- `poly` に、最終 IR と推論中 state を混ぜないための crate / type doccomment を追加した。
  - `poly::expr`: Arena-local ID、Def / Expr / Select / Pat の責務。
  - `poly::types`: polarity 分離、TypeArena、TypeVar、SubtractId、Scheme の責務。
- `infer` に、作業 table / machine の設計意図を doccomment として追加した。
  - `infer::Arena`: fresh ID と constraint machine の所有。
  - `typing`: `Computation` と DefId 型 table の境界。
  - `uses`: Ref / Selection use-site meta と watcher の境界。
  - `constraints`: weighted subtype、bounds 伝播、subtract fact の境界。
  - `analysis`: constraint / selection / SCC の event routing。
  - `scc`: incremental component graph、method dependency、量化で削除する flow。

### Resolver order lookup footing

- `infer::ModuleTable` を `HashMap<Name, DefId>` から、module 内宣言列 + 名前 index へ変更した。
  - `ModuleOrder` は source 上の宣言順。
  - `ModuleValueDecl` / `ModuleChildDecl` は name / visibility / order / target を保持する summary。
  - `AliasDecl` も order を持つ。import view の不動点構築は次段階だが、source 位置は失わない。
- lookup API:
  - `value_at(module, name, site)` / `module_at(module, name, site)`:
    同じ module の中で、`site` 以前の同名宣言があれば最後を選ぶ。無ければ `site` より後の直近を選ぶ。
    binding body は header 後を解決しているため、同じ order の宣言も見えている扱いにする。
  - `lexical_value_at` / `lexical_module_at`:
    current module で見つからなければ parent module へ上がる。その時の site は
    「子 module 宣言が親 module に現れた order」に変換する。
- regression test:
  - previous shadowing が勝つケース。
  - previous が無い場合に forward ref の直近が勝つケース。
  - child module から parent を見る時、child module 宣言 order を基準にするケース。

確認:

- `cargo fmt -p infer -p poly --check`
- `cargo test -p poly -p sources -p infer`（sources 3 passed / infer 27 passed、poly 3 passed）
