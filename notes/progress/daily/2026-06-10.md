# 2026-06-10

## Act resolution strict type reader

- `infer::act_resolve` を追加し、act が参照する strict type expression を `AnnType` から分離した。
  - 読む形は `Builtin` / `Named` / `Var` / `Apply` / pure `Function` に限定。
  - effect row / effectful annotation / wildcard など、annotation 固有の surface は受け付けない。
  - `resolve_act_use` は head が `ModuleTypeKind::Act` に解決できる場合だけ成功する。
- `act local = source ...` の copy 元解決を `ActTypeResolver` 経由に変更した。
  - copy 自体は act resolution の利用者として扱う。
  - source act の型変数に対して、引数が bare type variable のときだけ alias を登録する。
  - `source (a -> b -> c)` のような strict function type argument は解決できるが、alias にはしない。

確認:

- `cargo check -q -p infer`
- `cargo test -q -p infer act_resolution_ -- --nocapture`
- `cargo test -q -p infer resolves_act_copy_type_arg_aliases -- --nocapture`
- `cargo test -q -p infer`
- `cargo test -q -p yulang-parser`
- `cargo test -q -p yulang2`
- `cargo check -q -p yulang-sources`
