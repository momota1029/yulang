## Act resolution strict type reader

- `infer::act_resolve` を追加し、act が参照する strict type expression を `AnnType` から分離した。
  - 読む形は `Builtin` / `Named` / `Var` / `Apply` / pure `Function` に限定。
  - effect row / effectful annotation / wildcard など、annotation 固有の surface は受け付けない。
  - `resolve_act_use` は head が `ModuleTypeKind::Act` に解決できる場合だけ成功する。
- `act local = source ...` の copy 元解決を `ActTypeResolver` 経由に変更した。
  - copy 自体は act resolution の利用者として扱う。
  - source act の型変数に対して、引数が bare type variable のときだけ alias を登録する。
  - `source (a -> b -> c)` のような strict function type argument は解決できるが、alias にはしない。

確認:

- `cargo check -q -p infer`
- `cargo test -q -p infer act_resolution_ -- --nocapture`
- `cargo test -q -p infer resolves_act_copy_type_arg_aliases -- --nocapture`
- `cargo test -q -p infer`
- `cargo test -q -p yulang-parser`
- `cargo test -q -p yulang2`
- `cargo check -q -p yulang-sources`
- `cargo fmt --all -- --check`

## Constructor NegSignature

- struct / enum constructor payload の型を `AnnType` ではなく `NegSignature` として読むように変更した。
  - data representation は反変側の signature として扱い、payload 変数へ `value <: signature` を置く。
  - owner type variable は constructor の owner 引数と共有する。
  - field の `() -> [ref_update 'a; 'e] ()` のような返り effect row は、negative lowering で subtract fact まで登録する。
- テスト fixture は binding annotation からではなく、実際の `struct` field から `TypeExpr` を取る形にした。

確認:

- `cargo test -q -p infer neg_signature_builder_preserves_effect_row_tail_for_data_representation -- --nocapture`
- `cargo test -q -p infer struct_constructor_field_signature_records_subtractability -- --nocapture`
- `cargo check -q -p infer`
- `cargo test -q -p infer`
- `cargo fmt --all -- --check`

## Struct Field Selection

- struct field から synthetic type method を採番するようにした。
  - `x.field` 用に value method を作り、`T -> Field` の scheme を直接制約として置く。
  - `&x.field = value` 用に ref method を作り、`ref 'e T -> ref 'e Field` の scheme を直接制約として置く。
  - synthetic method は通常の値名前空間へ登録せず、selection の method table だけへ入れる。
  - 同名の明示的な type method がある場合は、selection 候補としては明示 method を優先する。

確認:

- `cargo test -q -p infer struct_field_lowers_to_value_method_selection -- --nocapture`
- `cargo test -q -p infer struct_field_ref_assignment_lowers_to_ref_method_selection -- --nocapture`
- `cargo check -q -p infer`
- `cargo test -q -p infer`
- `cargo test -q -p yulang2`
- `cargo fmt --all -- --check`

## `std::control::var::ref.update` lowering

- `lib/std/control/var.yu` の `ref::update` が `<missing>` になっていた原因を切り分けた。
  - type / act / role / impl method lowering が receiver 後ろの header 引数を lambda 化していなかった。
  - `my loop(x: [_] _) = ...` の `_` value annotation が未対応で、型名 `_` として解決されていた。
  - local binding body を lower してから local slot を登録していたため、local self recursion の `loop` が見えなかった。
  - `ApplyColon` が expression lowering 未対応だった。
  - `std::control::var::ref` の selection が常に payload 側を先に見ていて、`ref` 型自身の synthetic field method を見逃していた。
- 修正後、`std.control.var.ref.update` は scheme/body を持ち、`r.update_effect()` は synthetic field method `update_effect` へ解決される。

確認:

- `cargo test -q -p infer type_method_header_arg_lowers_after_receiver -- --nocapture`
- `cargo test -q -p infer std_ref_update_method_body_lowers -- --nocapture`
- `cargo check -q -p infer`
- `cargo test -q -p infer`
- `cargo test -q -p yulang2`
- `cargo fmt --all -- --check`
- `git diff --check`
